Update Settings

Updateable

Specify if the field can be edited. TIP: If you want to be able to type text in the field, but still not update any table when the Grid is submitted, set 'Updateable' to true, and set the 'Table Alias' property to: *unbound.

Blank is NULL

Treat blank values as NULL. IMPORTANT: You can only use this option if your database allows this field to be nullable.

Security level

Specify the minimum Security Level required to update this field. If all users can update this field regardless of their Security Level, set the Security Level to 0.

Security Groups

Select which security groups can update this field. If no groups are selected, all users can update this field.

Table alias

Specify the table alias for the field you want to apply the update to. TIP: If this field is not bound to a table, specify: *unbound (i.e. an asterisk followed by 'unbound')

Field to update

Specify the name of the field you want to apply this update to.

Check for write conflicts

When updating records, if 'Check for write conflicts' is turned on, should this field be included in the list of fields that are checked for write conflicts? (Certain SQL field types - e.g. 'ntext' - must be excluded.)

Geography field info (longitude,latitude)

When an update, or insert is performed, the location value in this field can automatically be computed from the longitute and latitude value from the fields that contain these values. If you leave this field blank, then this field will not be automatically updated when you insert or update a record. If you want to automatically update this field, enter the longitude and latitude field name (comma delimited). For example: myLongitudeField,myLatitudeField
