Map properties

Proximity search properties

Specify the proximity search control properties.

Control Javascript

Specify any Javascript code that should be emitted for this control. You can use the special {Grid.ControlId} and {Grid.ControlIdEscaped} placeholders to reference the control's Id. This is an advanced feature typically used by developers who are using a 3rd party Javascript library, such as JQuery, to style a control using Javascript.

Search style

Specify the type of search to perform. 0 = Use default setting specified on Search Properties tab, 1 = 'Exact Match' (may or may not be case sensitive - depends on the database), 2 = 'Is Contained In', 3 = 'Starts with' 4 = 'Wildcard' - user must explicitly enter wildcard characters '*', '%' or '?'. (If QBE syntax is turned on you can enter multiple wildcard criteria, delimited with either a comma or && - use && to combine criteria with an AND clause) For databases that are case-sensitive, you can use 11, 12, 13 or 14 for case-insensitive versions of the 1, 2, 3, or 4 option. You can use negative values (e.g. -1,-2,-3 or -4 to invert the search result (e.g. 'not equals'). For multi-valued fields (e.g. Checkbox and Dropdownbox controls that allow multi-select), 20=combine selections with OR, 21=combine selections with AND. For example, if a Checkbox control has 'Red', 'Green' and 'Blue' as its choices, if you check 'Red' and 'Green' and select option 20, the search is for 'Red' or 'Green' in the field, and if you select 21, the search is for 'Red' and 'Green' in the field. Instead of specifying 20, you can specify 201, 202 or 203. These all combine selections with OR, but allow you to control whether to search for an exact match (201), an 'is contained in' search (202, which is same as 20), or 203 a 'starts with' search.

Search expression

Specify the expression to search.

Range search

Display two textboxes to prompt for a starting and ending value. Allows user to search for a range of values without having to use Query-by-Form syntax.

'From' label

Specify the label that appears above the first search field. This is the search field that represents the start of the range.

'To' label

Specify the label that appears above the second search field. This is the search field that represents the end of the range.

Initial value

Specify the default value for this control when the Search Part is rendered. If this is a 'Range Search' enter two values, separated by || (two 'pipe' characters). For example: =date()||=date()+5 TIP: To enter an expression, enter a leading = sign. For example: =date(), =now() (today's date and time). You can use session variables as default values. For example: session.companyName. You can use Page Variables (i.e. the variables defined in the URL) by prefixing the variable name with 'PageVariable.' (e.g. PageVariable.lastname)

Keyword search

Specify if this field is a keyword search. If so, then instead of searching in this field, Alpha Anywhere will search for a match in any of the keyword fields.

Keyword search fields

Specify which fields to search when the user enters search criteria into this field.

Lookup type
