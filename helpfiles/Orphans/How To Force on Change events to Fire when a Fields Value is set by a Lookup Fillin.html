How To Force onChange events to Fire when a Field's Value is set by a Lookup Fillin

A user sent in a bug report saying:

1. he has a field (called say 'FIELD1')
2. he has defined an onChange event for FIELD1
3. he has also defined FIELD1 to be a lookup field.

If he types a value into FIELD1, then of course the onChange event fires.

But, if he clicks on the icon to open the lookup window and makes a selection from the Lookup Window, which fills in a value into FIELD1, the onChange event does NOT fire.
He wants it to fire.

The reason it does not fire is that if you get a pointer to an input control in an HTML page, and then set the .value property of that element, the onChange does not fire. This is just how Browsers behave. We have no control over this.

And that's exactly what is going on behind the scenes when you make a selection from a Lookup Window - we are getting a pointer to the fields that have to be filled in and then setting their value property.

So, here is how you can force the onChange to fire even though the user did not type a value into the field:

1. go to the 'afterLookupGridFillinValues' client-side system event and add this code

rowNum = {Grid.object}._selectedRow;
ele = $('{grid.componentName}.V.R' + rowNum + '.FIELD1');
ele.onchange();

Note: If a value in a Grid is changed using the .setValue() method, the onChange (and all other events) do fire.
