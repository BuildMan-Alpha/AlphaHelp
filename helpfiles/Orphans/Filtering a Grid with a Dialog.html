Filtering a Grid with a Dialog

One of the most common tasks a web application can perform is to collect the values of one or more variables, then use this data to display a filtered view of a table.

Creating a Grid Component for the Alpha Five Customer Table

  1. Create a grid component based on the AlphaSports Customer table.

  2. Display the Grid > Query (DBF) page:

    1. Click next to the Order field to display the Order Builder.

    2. Click to select "Lastname" in the Sort by list.

    3. Click to select "Firstname" in the then, sort by list.

    4. Click OK to continue.

  3. Display the Grid > Fields menu.

    1. Select several fields, including the Bill_state_region field.

    2. Fix the Column Properties > Column headings to make them attractive.

  4. Click File > Save as and name the grid component "Customers".

  5. Close the Grid Component Builder.

  6. Create a new A5W page.

  7. Click File > Save as and name it "Customers".

  8. Click Insert Component to display the Insert Component dialog.

  9. Click Select, pick the "Customers" component, and click OK.

  10. Add a title and center the page elements.

  11. Save the page as  "Customers" and exit from the HTML Editor.

  12. Click and the result should look something like this:

Setting a Variable in a Dialog Component

The next step is to create a dialog component with a drop down list box control. The control is populated with a list of the unique state values (from the Bill_state_region field) found in the Customer table.

  1. Create a dialog component with a drop down list control.

  2. Display the Form > Controls menu.

  3. Set Control Settings > Name to "States"

  4. Leave the Control Settings > Scope as "Local".

  5. Click in the DropDownBox Properties > Choices field.

    1. Define Choices > List Type = "Dynamic"

    2. Define Choices > Maximum Choices = 0.

    3. In the Data Source > Table name field select "customer".

    4. In the Data Source > Table name field enter "Bill_state_region".

    5. Click OK to continue.

  6. Set DropDownBox Properties > Height to 6.

  7. Display the Form > Properties menu.

  8. Click to set Server Events > AfterValidate to the following expression.

CurrentForm.redirectTarget="Customers.a5w?customers_Filter=Bill_state_region=" + quote(states)

  1. Click File > Save as and save the dialog component as "States".

  2. Create a new A5W page.

  3. Click File > Save as and save the page as "States".

  4. Click Insert Component to display the Insert Component dialog.

  5. Click Select, pick the "States" component, and click OK.

  6. Save the new page and exit from the HTML Editor.

Understanding the Dialog After Validate Event

Previously we set the After Validate of the dialog component to the following expression.

CurrentForm.redirectTarget="Customers.a5w?customers_Filter=Bill_state_region=" + quote(states)

The CurrentForm.redirectTarget command tells the Application Server to next display a different page. The page to display is Customers.a5w. The command uses the A5W Page Filter Syntax to specify that the Customers page must apply the filter Bill_state_region=states, where Bill_state_region is a field in the Customer table and states is the name of the drop down list box control and variable of the dialog box. So, if you selected "DC" in the drop down list box control, and clicked Submit, the expression would be:

CurrentForm.redirectTarget=Customers.a5w?customers_Filter=Bill_state_region="DC"

  1. Select the "Customers.a5w" page in the Web Projects Control Panel, right click, and select Publish (Local Webroot) and open.

  2. Select the "States.a5w" page in the Web Projects Control Panel, right click, and select Publish (Local Webroot) and open.
