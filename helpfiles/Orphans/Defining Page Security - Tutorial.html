Defining Page Security - Tutorial

The third step when implementing the web security framework is to define which of your pages will fall into these three categories:

    • pages that web users can never open (Always Denied)

    • pages that web users can open without login (Always Allowed)

    • pages that web users can open after successful login (Login Required)

  1. Display the Web Projects Control Panel.

  2. Click Security to display the Web Security dialog.

  3. Select Page Security and click OK to display the Page Security Assignment dialog. This dialog initially lists all your pages except the Login.A5W page in the left-most Always Denied column.

  1. Select all the entries in the Always Denied column and click Move > Move Selected to Login Required.

  2. Select "Accounting.a5w" and "Private.a5w" in the Login Required list.

    1. Click Set Permissions for Select Pages.

    2. Check Accounting.

    3. Click OK.

  3. Select "Marketing.a5w" and "Private.a5w" in the Login Required list.

    1. Click Set Permissions for Select Pages.

    2. Check Marketing.

    3. Click OK.

  4. Click OK.

Limitations

Web publishing applications only.
