About Validation

Ideally, Validation() methods validate that tables and columns actually. Depending on the database they may also return errors if the user does not have the right to perform the operation.

Validation involves a number of things:

  1. Syntax, which is adherence to a particular format.

    1. The AlphaDAO parser evaluates portable syntax and then hands the parse tree of to a syntax handler to generate the SQL for the target database.

    2. A syntax error in the case of Validate would imply a bug in the syntax handler

  2. Semantic checking of contextual items.

    1. Omitting a column that cannot be NULL from an insert statement.

    2. References to tables or columns that do not exist.

    3. Literals and column references that are incompatible with the types of the value being assigned or compared with or the type of the function argument they are passed for.

    4. Function names that are incorrect

    5. The wrong number of arguments for a function.

  3. Access permissions

    1. Users can be denied access to tables for write or for read. A good database engine would check this in the validation phase.
