Different Types of Objects in Alpha Five

In Exercise 6, you explored an Alpha Five object of type form. You saw that a form object has certain properties and methods and events that characterize the behavior of a particular instance of a form (like the Sign In form). So when you changed the value of the Sign In form property background color, the appearance of the Sign In object changed. We also saw that Alpha Five forms have events that can be defined with Xbasic scripts. When an event happens, Alpha Five performs the Xbasic commands in that events script.

Keep in mind that Alpha Five includes many different types of objects. All of these objects are components of Alpha Five. Objects include: data tables, forms, browses, controls, the Alpha Five program itself, the Code Editor, the Control Panel, etc. All of these objects have properties and methods. And some types of objects respond to events if you write an Xbasic script for those events.

Some objects are always present in Alpha Five whether you are using Alpha Five interactively or with a pre-built application. These objects are called system objects.  Examples of system objects are Control Panel, Status Bar, Form, Browse, Set, Table, and the Alpha Five program itself.

A system object is an object that is built-in to Alpha Five. You can't create new objects that have the same characteristics as system objects.

But system objects (which allow you to control the behavior of the Alpha Five program) can be referenced and controlled (for example opened and closed through Xbasic scripts.

For example, in exercise 6, the Code Editor that you opened to view the Xbasic commands of the OnActivate event is a system object. There is only one "instance" of an Alpha Five system object. The Code Editor is the instantiation (or instance) of the Code Editor system object.

Objects that you create instances of are based on the Table, Set, Operations, or any of the various Layout objects.

An instance of an object is an occurrence or implementation of that object type. A Rolls-Royce Cloud is an instance of a car object. The Sign In form is an instance of a form layout object.

For example, if you create a new database and define a new table named clients, then Alpha Five creates a table type object named clients that has the same available characteristics (properties and methods) as the system Table object. Likewise, when the Sign In form was created for the Phone Messages database application, that object was based on the system form layout object.

The technical term for creating your own object from an object type is called instantiating that object. The Sign In form layout object is an instantiation of the form layout object type. You can create tables and sets, browses, forms, reports, labels, and letters and operation type objects in Alpha Five.

Alpha Five also instantiates certain system objects itself, but you cannot create additional objects of that type yourself. (For example, you cannot create additional Control Panel objects). But you can manipulate the Control Panel system object through its properties and methods.

A specific object type is also known as a class or category of objects. So a particular form layout object can be spoken about as belonging to the form layout class of objects.

Most of the objects that you will use in Xbasic scripts will be layout and table objects (or simply layouts and tables). In Exercise 6, you used Alpha Five to change the Sign In form object's background color property through the Alpha Five properties window. In order to change an object's property or invoke an object method in an Alpha Five Xbasic script you must know how to reference that object using object addressing.
