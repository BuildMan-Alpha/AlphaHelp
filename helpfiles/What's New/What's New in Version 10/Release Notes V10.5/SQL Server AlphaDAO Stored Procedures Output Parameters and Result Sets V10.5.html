SQL Server AlphaDAO Stored Procedures Output Parameters and Result Sets

SQL Server updates binds parameter values only after all result set rows have been fetched from all result sets.

If you have no input/output or output parameters, no changes are required. Your application will continue to work as previously.

If you AlphaDAO command has input/output or output parameters, but there are no result sets returned from the stored procedure, values are available immediately after executing procedure. Your application will continue to work as previously.

If you are calling a stored procedure that returns both one or more result sets and input/output parameters, you need to fetch all result set values and explicitly call SQL::ResultSet::NextResult() on the active result set to enable retrieval of updated input/output and output parameters.


For example:
Dim Connection as SQL::Connection
Dim Arguments as SQL::Arguments
 
'� code to connect and execute a query �
 
ResultSet = Connection.ResultSet
 
 
'� code to fetch all result set rows for all result sets �
 
ResultSet.NextResult()
 
'� Arguments will now contain updated input/output and output parameters.
