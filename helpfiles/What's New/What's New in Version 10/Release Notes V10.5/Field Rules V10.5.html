Field Rules V10.5

Field Rules - Saving

Saving Field Rules after you have made an edit is now dramatically faster. This is especially evident in tables with a large number of fields.

Field Rules - Unique Value Field Rule - Blank Values

A change has been made so that when you set a field that previously had a non-blank value to a blank value, the Unique value field rule is no longer applied. This now allows you to set a field to a blank value when there are already other rows that have blank values.

Field Rules - UNC File Names

If a lookup table in Field Rules was defined using a UNC filename (e.g. \\mydata\alphasports\vendor.dbf) and if the UNC folder was mapped to a drive letter (e.g. \\mydata mapped to drive 'P'), then if a database was opened using a mapped drive letter rather than the full UNC name, lookups would fail. (e.g. user open p:\alphasorts\alphasports.adb rather than \\mydata\alphasports\alphasports.adb)
