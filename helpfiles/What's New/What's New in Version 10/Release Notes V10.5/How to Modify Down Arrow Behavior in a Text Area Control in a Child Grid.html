How to Modify Down Arrow Behavior in a Text Area Control in a Child Grid

In a Grid component, pressing the down or up arrow causes focus to move from one row to another. The exception is when a Text area control has focus. In this case the up and down arrows move from line to line in the Text area control. However, if the Grid was opened from a button on a parent Grid, then when the Text area has focus, up and down arrows cause the focus row in the parent Grid to change (and not the focus row in the Text area, as you would expect).

The solution to the problem is to add this Javascript (using the In-line Javascript action) to the Javascript that opens the child Grid:
$e.stopEvent(event); return false;
