Recommended Design Pattern for Using Client-side Xbasic


If you are building a Grid that you intend to use both on the Desktop and on the Web the following design pattern is recommended when you want to use client-side Xbasic on the desktop.

  1. Define the function to handle the event using Javascript.
  2. Inside this Javascript function use the new ._embeddedMode property of the Grid Object to determine if you want to handle the event using Javascript or Xbasic.

The Grid Object's _embeddedMode property is set to true when the Grid is running on the Desktop or in Working Preview mode. It is false when the Grid is running on the Web.

For example, the Javascript function that you define to handle an HTML event might be:

function myFunction(obj) {
    if({Grid.object}._embeddedMode) {
        myXbasicFunction(obj);
        return false;
    }
 
/*
continue here with the Javascript to handle the event when the Grid is running on the Web
*/
alert('This is javascript - you clicked on a button called ' + obj.id);
}


You can generally pass the same arguments to the Xbasic function that you pass to the Javascript function. Here is how the Xbasic function that the Javascript calls might be defined:

function myXbasicFunction as v (obj as p)
dim msg as c 
msg = "This is Xbasic - you clicked on a button called " + obj.id
ui_msg_box("Notice",msg)
end function
