How to make the code that hides a column more robust

If you hid columns in override settings, then the sorting on columns after the hidden column would be messed up.

For example, the following code in the Override Settings section on an .A5W page hides column 2 on the Grid:

tmpl.field_info[2].column.hide = .t.
'It is necessary to recalc all grid properties when you hide a column
tmpl.flagMustRecalculateAllProperties = .t.


Tip: Here is how the code that hides a column can be made more robust so that it does not rely on an array index, but instead, relies on the column name:

dim indx as n
indx = tmpl.field_info.find("LASTNAME","ut(fieldname)")
if indx &gt; 0 then
    tmpl.field_info[indx].column.hide = .t.
end if
tmpl.flagMustRecalculateAllProperties = .t.
