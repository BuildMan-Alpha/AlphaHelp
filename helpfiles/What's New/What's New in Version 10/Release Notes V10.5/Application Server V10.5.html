Application Server V10.5

Server.SessionCount Property

This new read-only property of the Server object indicates the number of active (unexpired) sessions on the server.

Session.Abandon() Method

Abandons the current session, deleting the session folder (along with its contents) and all session variables. The session and session folder will continue to exist for the remainder of the current A5W page execution, but any changes made will not be persisted and the session will be fully removed when execution of the current page is complete.

Session.IsNewSession Property

This read-only property indicates whether or not the current session was just created, or if it already existed at the time that the current request was received by the server. Returns True if the session is new, False otherwise.

Throttling Sessions

It is now possible to throttle the number of simultaneous sessions handled by a server using the newly introduced functionality. An example implementation would be to use an a5w_include() statement on each page of your application with code such as the following:
<%a5
'Limit the number of simultaneous Sessions on this server
dim MaxSessions as n = 5
If Session.IsNewSession .and. Server.SessionCount > MaxSessions
    'delete this newly created session which exceeds the desired threshold
    Session.Abandon()
    
    'Display a message to the user
    ? "<html><body>The server is too busy to process your request. Please try again later.</body></html>"

    'stop the page execution
    end
end if
%>

Search Engines

The Application Server now generates a unique session tracking identifier for search engine spiders that do not support cookies. Previously a constant value of "search_bot" was always used across server starts, which could potentially introduce unintended session tracking behavior for search engine spiders.

a5w_info() Function

Now displays Aliases defined in your web application for help in debugging.
