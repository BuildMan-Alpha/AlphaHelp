Grid Component - Language Translation - International Characters

The Grid component allows you to define language strings to automatically translate any string that is coded with the <a5:r> and </a5:r> language tags.

If the translation string contained an international character (e.g. 'Prénom'), then the translated Grid had included a garbage character. This is now fixed.

However, the fix will impact users who do not define the translation string in the Grid itself, but instead define a database query in the in the Grid's onGridInitialze event to retrieve the translation string.

In those cases, if the translation string contains international characters, you will need to set a new property of the Grid in your onGridInitialize event:
e.tmpl.encodeLanguageStringAsUTF8 = .f.

This property will prevent the Grid from encoding the translation string as UTF8 because it will already be in UTF8 format.
