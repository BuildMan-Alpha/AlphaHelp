Grid Component - Client-Side Calculated Fields - Circular References

If you define a client-side calculated field with a circular reference, the builder now warns you that you have made an error. For example, assume you have a field in your Grid called 'Total' and you define this client-side expression:
Total = 'Amount Due: ' + Total

This is a circular reference. The first time the calculation is performed, the result will be:
Amount Due: 23.45

The next time is is performed, the result will be:
Amount Due: Amount Due: 23.45

This is clearly not what anyone would want. The expression contains a 'circular reference' because the field for which the expression is defined is included in the expression.
