Reports - SQL Data Sources

When you print a report that is based on a SQL data source you can specify the filter in two places. The <layout>.print(), <layout>.saveAs() and <layout>.preview() methods take a 'filter' argument. This filter is a .dbf filter that is applied after the data has been retrieve from the server (inefficient). You can also pass in a dot variable for the optional 'Options' parameter. This dot variable can have a .filter and .order properties. These are SQL filter and order properties that are evaluated by the server before any data is sent to Alpha Five (extremely efficient).

Previously, there was no way to print out the value of the SQL Filter and Order on reports. Now, two new functions make this easy. You can define calculated fields using these functions:

a5_getReportSQLFilter(L flagResolve)
a5_getReportSQLOrder()

If the flagResolve argument is .t. (its default value), then any arguments in the Filter are resolved. For example:

lastname = "Smith" and balance > 1000

If the flagResolve argument is .f., the arguments in the Filter are not resolved. For example:

lastname = :whatLastname and balance > :whatbalance

This new feature is particularly useful in the case of Web Applications because the Action Javascript Genies always generate code that use SQL filters (and not local .dbf filters).
