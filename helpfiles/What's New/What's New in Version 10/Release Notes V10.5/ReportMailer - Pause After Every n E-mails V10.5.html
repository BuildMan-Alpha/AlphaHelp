ReportMailer - Pause After Every n E-mails

A new version of ReportMailer has been released. If you have purchased the Reportmailer add-on, you can upgrade your copy by downloading it again using the same URL that you used when you downloaded your original copy. This update allows you to specify a 'batch size' and then pause for a specified number of seconds between batches. This is necessary because if you send too many e-mails too rapidly, some ISPs will consider the e-mail to be SPAM, and will block it.

The following two new properties can be added to your Xbasic code beforre you call the ReportMailer function

distribute_params.pauseAfterHowMany = 3
distribute_params.pauseHowLong = 10

The 'pauseAfterHowMany' property is the number of e-mails to send before pausing. If this property is set to 0, then ReportMailer will not pause.

The 'pauseHowLong' property indicates how long to pause, in seconds.

The ReportMailer genie and ReportMailer Action Script automatically prompt for these two new properties.
