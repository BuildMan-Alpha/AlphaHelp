Dynamic Connection Strings - Reports, Labels and Letters

In build 3420-3606 support was added for dynamic connection strings to allow developers to build multi-tenant web applications where each tenant had their own connection string. However, dynamic connection strings were not supported in Reports, Letters and Labels until this update.

For example, say you have built a report called 'CustomerReport' in your application. The connection string for this report was defined as:

::name::DynamicConnection_conn1

In your Publishing profile, 'DynamicConnection_conn1' is might be defined as follows:

{A5API=Access,FileName='C:\Data\Customer1.mdb',UserName='Admin'}

When the report is run, it will be based on data in 'C:\data\Customer1.mdb'.

In your web application, you could have an .a5w page that executes this code:

session.__protected__conn1 = "{A5API= Access,FileName='C:\Data\Customer2.mdb', UserName='Admin'}"

After the above .a5w page had been run, when you then run the report, the 'DynamicConnection_conn1' gets resolved and the report is now based on 'c:\data\Customer2.mdb'.

You can use dynamic connections in Desktop Applications as well.
For example, you can execute this code to change how 'DynamicConnection_conn1' is resolved:

dim g as p
g = global_variables()
g.session.__protected__conn1 = "::name::someConnectionString"
