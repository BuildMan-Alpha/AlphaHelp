Grid Component - Entering Data Values

User Defined Switch Year For 2 Digit Year Values

By default, if you enter a 2 digit year in a date field, the switch year is set to 20. This means that any year value less than or equal to 20 is assumed to be a '2000' date, and any year value greater than 20 is assumed to be a '1900' date. You can now define the switch year. To change the switch year, you must override the Grid's '.clientSideDateFormat' property.

You can override Grid properties in many places - e.g. in the .a5w page that runs the Grid, or in action Javascript when you open a Grid, or in the Grid's onGridExecute event.

In the case where you want to set the switch year, it is easiest to override settings in the OnGridExecute event.

The following example shows the syntax for overriding the switch year in the OnGridExecute event. In the example, the switch year is set to 80.

e.tmpl.clientSideDateFormat = "MM/dd/yyyy||80"

Notice that the two pipe characters separate the date format and the switch year.

Update: As of build 3345_3595, the Grid has a new property that allows you to define the switch year without having to use Xbasic, although the Xbasic given above will still work.

Note: This switch year property and the date picker switch year property are complementary. This one applies to typed dates.

Entering Date Values without a Year Part

You can enter data values without a year part. If you leave out the year, the current year is assumed. So, for example, if you data format is mm/dd/yyyy and you enter '7/19', the date is assumed to be 7/19/2010

Restrictions

The features discussed above require V10.5 Build 3209_3580 or later
