AlphaDAO - A5InitialCommand Property

A new property has been added to the AlphaDAO connection string that allows you to execute a SQL command as soon as you connect to the target database.
You can also set this property programmatically (before the connection is opened). The SQL::connection object has a new read/write property call 'InitialCommand'.

At this point, only Connection String builder for ODBC connections has been modified to allow you to set the A5InitialCommand property. At some point, the Connection String builder for other connection types will be updated.

For example, say you wanted to execute this SQL command when you connect to the database:

SET TEMPORARY OPTION connection 
  authentication='Company=MyCo; Application=MyApp; Signature=0fa55159999e14d818e...'

The connection string for the database would then be:

{A5API='ODBC',DataSource='demodata',A5Syntax='SQLAnywhere',A5InitialCommand='SET TEMPORARY OPTION connection authentication=''Company=MyCo; Application=MyApp; Signature=0fa55159999e14d818e...'''}

Note that the SQL command appears in the connection string as A5InitialCommand='sql command goes here'.

Since the SQL command is enclosed in single quotes in the connection string, single quotes within the command itself must be escaped (by replacing ' with '' (a double single quote).
