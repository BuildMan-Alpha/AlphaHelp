Dynamic Report Group Break Expression - *current_expression() Function

You can now define a report that uses dynamic group break expressions. This will allow you to create a single report and then prompt at run-time, when the user wants to print the report to see if the user wants the report to be grouped on (say), 'Lastname', 'Company', or 'City'.

To support dynamic report grouping, a new function *current_expression() has been introduced. This function acts like the eval() function, but it is evaluated at Query time.

Here is how you could set up a report with a dynamic group break.
1. Before the report is printed, prompt for a Global variable. E.g.

dim global myBreakExpression as c= "state"

'state' is the name of a field in the table that the report is based on.

2. In the report, define the Group Break expression as follows:

*current_expression(var->myBreakExpression)

Before you print the report again, if you reset the value of the myBreakExpression global variable (say to 'Company'), when the report is printed, it will now be grouped on the 'Company' field.
