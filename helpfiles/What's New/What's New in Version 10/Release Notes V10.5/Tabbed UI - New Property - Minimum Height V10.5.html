Tabbed UI - New Property - Minimum Height

You can now specify a minimum height for the Tabbed UI 'Content Area'. The 'Content Area' is the area that stretches from below the 'Header' to above the 'Footer'. The default is 0 (no minimum height).

If the 'minimum height' is set to 0 (the default value), as you resize the browser window, reducing the window height, the TabbedUI is automatically adjusted so that the header and footer are shown and the remaining area is devoted to showing the pane contents.

However, say you set a minimum height of 200 (pixels). As you adjust the height of the browser window, the height of the 'Content Area' will never shrink below 200 pixels, and the window will show a vertical scrollbar so that you can scroll to the part of the TabbedUI that does not fit into the window.
