Web Applications - Security Framework

When you turn on the Security Framework in a new application, default values for some properties have been changed:
  • 'Logon Expiration Policy' is now set to 'Expires when current session expires'. If you change this value to some other policy a warning will be displayed to explain that you might be causing a problem in your application if you are relying on session variables for certain crucial data that is always assumed to be correctly set.
  • 'Enable component security for virtual pages' will be checked.
  • 'Always enforce security at component level for AJAX grids' ill be checked.
