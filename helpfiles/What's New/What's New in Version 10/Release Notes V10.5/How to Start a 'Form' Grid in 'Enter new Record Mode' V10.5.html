How to Start a 'Form' Grid in 'Enter new Record Mode'

A Grid can be configured to look like a Form and can also be configured to only allow entry of new records by setting the 'Add records only' property. If you set this property, the Grid does not allow editing of existing records - as soon as a new record is saved, focus moves to the next new record.

Some users have wanted to know how to configure a Grid so that it starts out in 'Enter new Record' mode (as it would if the 'Add records only' property is checked), but then, when the user saves the new record, the record just entered should remain on screen so that it can be edited (rather than focus immediately moving to the next new record as it would if you set the 'Add records only' property to true).

Here is how you can do this:

When you initially create the Grid use the 'Single Record Form' template. This template create a Grid with a Detail View and then hides the Grid Part. This creates a Grid that 'looks' like a form. The Grid will allow both entering a new records and editing of existing records. When the Grid is initially opened, it shows the first record in the current query.
In order to put the Grid into 'Enter new Record Mode' when it is first opened, go to the JavaScript - System Events property and then set the OnGridInitializeComplete event to this:

{Grid.object}.detailViewNewRecord();
