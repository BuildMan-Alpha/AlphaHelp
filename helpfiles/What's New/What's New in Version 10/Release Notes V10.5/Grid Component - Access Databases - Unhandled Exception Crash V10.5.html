Grid Component - Access Databases - Unhandled Exception Crash

On some machines — primarily those with Office 2010 installed — when you run a Grid that is based on an Access .mdb file, you will get an Alpha Five crash caused by an Unhandled Exception. The bug lies in the Microsoft Access Driver, not in Alpha Five, so there is nothing we can do to fix it. However here is an easy work around:
  1. Open the ODBC manager. If you are running a 64 bit operating system, you must open the 32 bit ODBC manager, which is typically called c:\windows\syswow64\odbcad32.exe
  2. Create a new DSN for your Access database. When you create the DSN you must specifically choose the older Access driver for .mdb files (Do NOT select the driver that supports .accdb files!)
  3. Create a new AlphaDAO connection string using the ODBC driver and select your Access DSN file.
