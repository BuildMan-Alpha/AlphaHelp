email_profile_add() Function

Email profiles are used to store email settings. Functions such as email_send() rely on email profiles. Email profiles are typically created through the User Interface by going to the View, Settings, Email command. Using the email_profile_add() function you can now create profiles using Xbasic.

The following Xbasic code shows how you might use this function 

dim em as p

'load an existing profile into a dot variable (a quick way to initialize

'all of the required properties


email_profile_load("Default", em)
em.mail_hostname = "mail.new_mail_host_name.com"
 

email_profile_add(em,"NewProfileName")
 

Tip: If your application sends e-mail, you might want to use Xbasic code that does NOT involve using profiles at all. This is especially true in Web applications where setting e-mail profiles is impractical. The following example shows how you can send email using low level Xbasic commands.

vTo = "addressee@mycompany.com"
dim pm as P
dim ps as P
dim email_result as l
vMessageStr = "here is may messsage."
flag = email_smtp_open(ps, "mail.mycompanymailserver.com", 25, "myuserid", "mypassword")
if flag = .f. then
        'report an error to the user
        end
end if
pm._to = vTo
pm.from = "me@mycompany.com"
pm.from_alias = "Your Name"
pm.subject = "Subject"
pm.attachments = ""
pm.html_message = "Here is my message"
email_result = email_smtp_send(pm, ps, .f., .f.)

 

if email_result <> .t. then

    'report an error to the user

end if
email_smtp_close(ps)
