Active-Link Tables - Builder - New Option for 'Where' Clause 

When you define an active-link table you can either allow Alpha Five to define the 'Where' clause used for updating records, or you can define your own Where clause. If you allow Alpha Five to automatically generate the 'Where' clause (this is the default option when you create an active-link table), there is a new option you can choose 'Primary Key and Modified Fields'. This new option is now the default. Previously, the default option was 'Primary Key and Updateable Fields'. The new option has been added because in some databases, if the table had a large number of fields, the Where clause that was generated was too long.
