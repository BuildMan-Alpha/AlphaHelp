Working with SqlTypes

The Microsoft SQL Data Types are installed with Alpha Five Version 11, and you can use them through the .NET integration layer to manipulate Geography data. The string and character types used here are SQL strings and characters and not standard .NET objects, so they don't map to normal Alpha Five strings.

' Given a well-known-text string
Dim MyText as C = "Point(1 2)"
 
' First convert the string to a SQLString.
dim s as System::Data::SqlTypes::SqlString = new System::Data::SqlTypes::SqlString(MyText)
 
' Create a Geography object
dim Geo as Microsoft::SQLServer::Types::SqlGeography
 
' Call the Geography object's Parse method to convert the String to an object.
g2 = geo.parse(s)
 
' Access the properties using the value keyword.  
' This is because the values themselves are SQL objects.
?g2.lat.value
= 2
?g2.long.value
= 1
