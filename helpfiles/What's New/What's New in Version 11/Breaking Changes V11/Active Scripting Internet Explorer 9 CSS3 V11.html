Active Scripting - Internet Explorer 9 - CSS3

The ability to respond to events in an HTML page using a language other than Javascript is referred to as 'Active Scripting'.

Unfortunately, starting with Internet Explorer 9 (IE9), Microsoft no longer supports Active Scripting, unless you set IE9 to operate in IE8 mode. However, if you select IE8 mode, then you can't use CSS3, and you have to give up effects such as rounded corners, gradients, etc.

Starting with this release, if you have IE9 installed on your computer, Active Scripting has been turned off by default. With Active Scripting turned off, that means you can no longer put Xbasic function definition into the HTML document in a <script> tag (with the language attribute set to 'Xbasic') and call these Xbasic functions (or any other global Xbasic functions) from HTML events, when the Grid is run in Working Preview mode.

However, if you want to forgo CSS3 support, you can turn on Active Scripting by going to the View, Settings command in the top menu, selecting Preferences, Active Scripting, Working Preview and selecting 'Yes'.

How to Handle Client-side Events with Xbasic Even When Active Scripting is Turned Off

With Active Scripting turned off, it is still possible to respond to an event in the component with Xbasic code as shown below. However, you cannot define the Xbasic code in the component - you can only call global Xbasic functions (i.e. User Defined Functions). In addition, your Xbasic event handler cannot see the DOM (Document Object Model) of the HTML document, as it can with Active Scripting. Depending on the application, this limitation could be quite serious.

Here is how you can respond to an event with Xbasic code:
  • Create a Javascript stub function that will handle the event.
  • Inside the Javascript stub function make a call to the CallXbasic() function. This function is a system Javascript function that is automatically loaded when a component is run in Working Preview.

For example, say you have a global Xbasic function called myXBCode() and you want to call this function from the onClick event of a button. Here is how you can define a Javascript function, called (say) jsCall_myXbCode() that will handle the onClick event:

function jsCall_myXbCode() { 
     var xBasicCommand = 'myXBCode()';
     callXbasic(xBasicCommand);
}

If the Xbasic function you are calling returns a value:

function jsCall_myXbCode() { 
     var xBasicCommand = 'myXBCode()';
     var returnValue;
     returnValue = callXbasic(xBasicCommand);
     return returnValue;
}
