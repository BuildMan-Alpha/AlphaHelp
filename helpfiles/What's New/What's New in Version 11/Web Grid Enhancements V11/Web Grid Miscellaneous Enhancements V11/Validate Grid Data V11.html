Validate Grid Data

Three new Grid methods have been added to validate Grid data. The data is submitted to the server, but is not actually saved. The field level validation rules are evaluated. The row level rules (e.g. canUpdateRecord, canInsertRecord, etc.) are NOT evaluated. These methods are ideal for implementing 'real time' validation as soon as the user tabs out of a field. You can call these events in the onBlur event of the control.

{grid.object}.validateGridRow(rowNumber)

{grid.object}.validateGrid()

{grid.object}.validateDetailView()

{grid.object}.validateGridField(rowNum,fieldName)
