Grid Component - New Property - Maximum Number of Records Per Page

If the number of rows per page in a Grid is set to 0, then the Grid will display all of the records in the query on a single page. If there are a very large number of records in the query this could tie up your server for a long time. The new 'Maximum Number of Records Per Page' property allows you to set a property to prevent too many records from being shown on a single page. For example, say you set the 'Rows' property to 0 (indicating that you want to see all of the records on a single page), and you run a Grid that has 1,000 records. If the 'Maximum Number of Records Per Page' has been set to (say) 200, then only 200 records will be shown on the first Grid page (despite the fact that Rows had been set to 0).
