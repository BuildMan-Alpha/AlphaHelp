Grid CurrentGridFilter() and CurrentGridOrder() Special Functions

These special functions are used in a Grid when you open a child Grid or Report and you want the child object to inherit the current Grid filter and order expressions. These functions assume (implicitly) that if the parent Grid is based on a SQL table, that the target Grid or Report is also based on a SQL data source, and similarly, if the parent Grid is based on a .dbf table, that the target Grid or Report is also based on a .dbf table.

However, in certain simple cases, the functionality of these two pseudo functions has been enhanced so that they will work across types (e.g. a SQL Grid that opens a report based on a .dbf table, or vice versa).

Here is an example of where the functions will NOT work. Say your Grid is based on a .dbf table and uses search option 2 (contains). You do a search for 'smith'. Alpha Five will generate this for the filter: smatch(lastname,"*smith*"). This obviously is not valid SQL syntax and so the currentGridFilter() function will not work when talking to an object based on SQL tables. However, if you change the search option to 1 (exact match), then Alpha Five generates this for the filter: lastname = "smith". This can be converted to SQL syntax by changing the double quotes to single quotes. E.g. lastname = 'smith'
