Geography Fields in the Grid - Searching

Say you have a table that has a Location field (a special geography field type supported by SQL Server, MySQL, Oracle and some other SQL databases, but NOT .dbf tables) and you would like to perform the following types of searches:
  • Find all records that are within a polygon defined by a set of location (latitude/longitude) co-ordinates
  • Find all records inside a circle defined by two location points (the circle center, and a point on the circle circumference)
To perform the first type of search, enter a comma delimited set of location points in the Search Part and use the prefix 'polygon:'. For example:

polygon:-70.877, 42.8125, -72.393, 42.6926, -72.32747, 42.0677, -70.6355, 42.0188, -70.877, 42.8125

To perform the second type of search, enter the center point location (longitude, latitude) and the radius (in meters). For example:

radius:-71.04, 42.34,14000

Notes about how the points are entered for the first type of search:
  • Points are longitude, then latitude
  • Points must describe a closed polygon (last point == first point)
  • Points must be counter-clockwise — the inside is on the left of the vector for each side
  • Polygon must be entirely contained in one hemisphere
  • Polygon should be open and not have crossed edges
Of course, as a practical matter, you cannot realistically expect a user to enter a search criteria by entering in a series of location points, so you will configure your Search Part to use a Map for the Location field, and the user can then describe the search polygon by clicking on the map to visually set up the closed polygon.
