Dynamic Filtering - Lookup Grids, Edit-combo Boxes, Auto-suggest Controls

When defining a dynamic filter for any of these control types (Lookup Grids, Edit-combo Boxes, and Auto-suggest Controls), the number of different operators that can be used in the dynamic filter has been expanded.

The new operators for the Lookup Grid are:
  • >
  • <
  • >=
  • <=
  • NotEquals
  • IsBlank
  • IsNotBlank
  • Wildcard
  • Equals any of
  • Starts with any of
  • Contains any of
  • Exclude all of

The new operators for the Edit-combo and Auto-suggest are:
  • Wildcard
  • Equals any of
  • Starts with any of
  • Contains any of
  • Exclude all of

The 'Equals any of', 'Starts with any of' and 'Contains any of' work as follows:

Say you have a Grid with a control called State. You have another control called Lastname and you have defined a lookup for this field that opens the Customer Grid. This lookup has a dynamic filter that references the 'State' field.

At run-time, say that the user enters 'MA,CA,NY' in the 'State' field and then clicks the lookup icon.

If the operator in the Dynamic filter is set to 'Contains any of', the lookup grid will show all records where the target field in the lookup Grid (say the 'Bill_State' field) contains 'MA', or 'CA', or 'NY'.

The 'Exclude all of' operator would show all records in the lookup Grid that are NOT MA, or CA or NY.

The wildcard operator allows the user to enter a value in the source field that contains the * or ? wildcard characters. For example, the user might enter '*ware*' in a company field, then click a lookup button that would show records such as:

'Alpha Software, Inc.'
'Beta Hardware Corp'
