AlphaDAO - MySQL - Transactions

Transaction support in MySQL has been improved. However, the following usage tip is important when using trancations:

Say you begin a transaction, and then, within the transaction you execute multiple INSERT, UPDATE and DELETE statements, but you also execute multiple SELECT statements and read data from the resulting result sets. After you are done with reach result set, it is very important that you delete the variable that reference the result set and that you call the .freeResult() method on the connection. You must also call the .freeResult() method before committing the transaction. The following psuedo code shows more information:

dim cn as sql::connection
cn.open("::Name::myconnection")
cn.BeginTransaction()
cn.execute(SQL Insert statement)
cn.execute(SQL Select)
dim rs as sql::resultset
rs = cn.resultset
dim data as c
data = rs.data(1)
delete rs 
cn.freeResult()
 
cn.execute(SQL Select 2)
dim rs as sql::resultset
rs = cn.resultset
dim data as c
data = rs.data(1)
delete rs 
cn.freeResult()
 
cn.execute(SQL Insert 2)
cn.execute(SQL Select 3)
dim rs as sql::resultset
rs = cn.resultset
dim data as c
data = rs.data(1)
delete rs 
cn.freeResult()
 
cn.freeResult()
cn.CommitTransaction()
