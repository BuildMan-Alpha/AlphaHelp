How to turn off the proportional vertical scroll bar

In version 9 we introduced 'proportional vertical scrollbars' in the browse and embedded browse.

Before v9, the vertical scroll bar was not truly proportional (except in the case where the records were ordered by record number)

With a true proportional scrollbar, the position thumb in the scrollbar is exactly proportional to the position of the record in the ordered set of records, and dragging on the thumb will accurately position you on the correct logical record in the ordered set of records.

However, computing the true position of the thumb in the vertical scrollbar is expensive. On a slow network, or with large tables, it might cause performance problems.

You can run a command that tells a5 not to use 'proportional' scrollbars by setting a command that says: if there are more than x records in the current query, use old style vertical scroll bar.

Note: In V11 (build 2488 or above), you can select the View, Settings command and then, in the Preferences setting, you can set the cutoff value.

For example:

This means ALWAYS use v9 proportional scrollbar:

a5.System_Mode_Set("index_pseudo_position_cutoff","0")

This means ALWAYS use v8 non-proportional scrollbar:

a5.System_Mode_Set("index_pseudo_position_cutoff","1")


This means use v9 proportional scrollbar if there are less than 50 records, otherwise use v8 non-proportional scrollbar:

a5.System_Mode_Set("index_pseudo_position_cutoff","50")
