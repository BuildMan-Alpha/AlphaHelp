How to end a string with a backslash

If you try to put a file path ending with a backslash into a string literal in Xbasic, you may have difficulty. That is because the backslash \ is an escape character when followed by a quote \"; the combination allows you to insert quote characters into string literals. Therefore, a string definition for a path such as:
dim mypath as c = "\mypath\"
is not valid Xbasic.

The cure is to add an extra backslash so that the second backslash is itself escaped:
dim mypath as c = "\mypath\\"

mypath will evaluate to \mypath\ as desired.
