How to make a grid component use a DBF table in a non-standard directory

1. Assume you have a table in your database that is NOT in the same folder as the .adb file. Let's assume that this table is c:\mydata\customer.dbf and that the .adb file is in a folder called c:\myapp

2. You then build a grid on this 'customer' table. When you save the Grid and you go to the Xbasic tab, you will see
tmpl.DBF.table_name = "c:\mydata\customer.dbf"

THIS IS A PROBLEM!!! The table name is not encoded using an alias.

3. Go to the View, Settings, command and then navigate to Aliases. Add a new alias. For example:

[Data2] --- c:\mydata

4. Go back to the Grid builder. Make the Grid dirty and resave it. Now go to the Xbasic tab. You will see that this property is now looking good:
e.g.

tmpl.DBF.table_name = "[Data2]\customer.dbf"

5. You can now go to your publishing profile and specify the location of the new alias, [data2]

Note that this technique also works for Access database files. It is not required for most SQL connection strings.
