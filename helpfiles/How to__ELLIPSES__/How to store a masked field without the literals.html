How to store a masked field without the literals

Q:

In working with fields and properties, I set a PhoneNumber to

ClientSideFormatting - Checked
FormatType - Mask
FormatDefinition - (000) 000-0000

Is there a way to not store the Mask Literals with the Data ?

A:

Every field has a 'format' property. If the 'format' property is non blank, then an 'unformat' property is shown.

The 'format' property refers to server-side formatting.

Even though you are using client side formatting, you must also use server side formatting to format the data for its initial display (the client side formatting only kicks in when you enter a new value in the field).

Here is what you must define for the 'format' property:

TRANSFORM(remspecial(<value>),"@R (999) 999-9999")

remspecial removes any formatting from the field before the transform() function applies the formatting. this will solve the display problem if by chance your database has some phone numbers that were formatted with parens, etc. and some that were not.

Then in the unformat expression you specify:

remspecial(<value>)

This removes any formatting before the data is written to the database.

If you get write conflict errors (you should not), then turning off write conflict checking for this field will solve the problem.
