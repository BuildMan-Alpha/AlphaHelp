Display Xdialog Form based on ADO/ODBC Data

Overview

Many applications require a view of external data contained in ADO or ODBC compliant databases. The Display Xdialog Form based on ADO/ODBC Data action allows you to quickly build a dialog box that reads and displays data from compliant databases, such as  MySQL, Oracle, and Microsoft Access.

Procedure

The Xdialog box needs a variable to capture the value generated by each control.

  1. Create a new Action Script.

  2. Select "Xdialog Windows" in the Category list.

  3. Select "Display Xdialog Form based on ADO/ODBC Data" in the Action list.

  4. Click OK to display the Script Genie.

  5. Select whether the data source is an Access .mdb file or an ADO Connection string.

  1. If you selected Access .mdb file  in step 1, continue with Access MDB File. Otherwise, continue with ADO Connection String.

Access MDB File

  1. Enter the MDB Filename and click Connect.

  2. Select whether to base the form on a Table or View or a SQL Select Statement.

  1. If you selected Table or View in step 2, continue with Table or View. Otherwise, continue with SQL Select Statement.

ADO Connection String

  1. Select Used Saved Connection String or Specify Connection String now.

  1. If you clicked Used Saved Connection String in step 1, make a selection from the Connection Name list. Otherwise, continue with Specify Connection String now.

  2. Click Connect.

  3. Continue with SQL Select Statement.

Specify Connection String now

  1. Click Build to display the Data Link Properties dialog box.

Note : See Windows documentation for a full description of the Data Link Properties dialog box. This procedure documented here works for Microsoft Access databases.

  1. Select Use Connection String and click Build... to display the Select Data Source dialog box.

  2. Enter a name for the data source in the DSN Name field and click New... .

  3. Select "Microsoft Access Driver (*.mdb)" in the Create New Data Source dialog and click Next >.

  4. Click Browse... and navigate to the folder that contains the Microsoft Access database.

  5. Select "All Files" in the Save as type field.

  6. Select the Microsoft Access database and click Save.

  7. Click Next >, Finish, and OK. to display the Select Database dialog.

  8. Select the Microsoft Access database and click OK.

  9. Select the resulting DSN file in the Select Data Source dialog box and click OK and OK to display the Data Link Properties dialog box.

  10. Click Test Connection.

  11. If the connection was successful, click OK.

  12. Select whether to base the form on a Table or View or a SQL Select Statement.

  1. If you selected Table or View in step 13, continue with Table or View. Otherwise, continue with SQL Select Statement.

Table or View

  1. Click to select a table or view in the Table/View name field.

  2. Optionally, enter a filter expression in the Filter field.

  3. Optionally, enter an order expression in the Order by field.

  4. Continue with More Fields.

SQL Select Statement

  1. Enter your SQL Select statement and click Test Select Statement.

More Fields

  1. Check the Form is read-only check box if you are not going to be writing to the table.

  2. If the Form is read-only check box is cleared, click Select to identify the primary key of the table.

  3. Click Next >.

  4. Select the fields you would like to display from the Available Fields list. Use to add and remove Selected Fields.

  5. Optionally, reorder the sequence of fields with .

  1. Optionally, select a field in Selected Fields.

    1. Optionally, modify its description in the Label field.

    2. Optionally, change its Control Type, Width, and Height.

    3. Optionally, change the Tab Page.

    4. Optionally, click Set All to set all remaining fields to this Tab Page.

  2. Click Next >.

  3. Select the buttons you would like to display from the Available Buttons list. Use , , , and to add and remove Selected Buttons.

  4. Optionally, reorder the sequence of buttons with , , , and .

  1. Optionally, select a button in Selected Buttons.

    1. Optionally, modify its Type. Click Set All to set this type for all remaining buttons. The options are:

      • "Text"

      • "Picture"

    2. Optionally, click the button image to select a different image.

    3. Optionally, select a different Border style. Click Set All to set this style for all remaining buttons. The options are:

      • "Raised"

      • "Modern"

      • "Fly-over"

      • "None"

    4. Optionally, change the default Bubble Help.

  2. Click Next >.

  1. Optionally, on the Window tab change the Window title.

  2. Optionally, change the Background color of the form.

  3. Optionally, change the Font and point size of the text on the form.

  4. Optionally, change the Button bar position. The options are:

    • "Bottom"

    • "Top"

    • "Top and Bottom"

    • "None"

  5. Optionally, check Enclose buttons in frame.

  6. Optionally, change the Form type. The options are:

    • "Modal"

    • "Modeless"

    • "MDI Child Window"

  7. If the choice in step 23 is "MDI Child Window":

    1. Optionally, select a Toolbar from the drop-down list.

    2. Optionally, select a Menu from the drop-down list.

    3. Optionally, change the Title bar icon.

  1. Display the Layout Options tab.

  2. Optionally, select a different Layout style. The options are:

    • "Form"
    • "Table"

  3. Optionally, check Repeating Form.

  4. If you checked Repeating Form:

    1. Enter the number of Rows.

    2. Enter the number of Columns.

    3. Select the Snaking option.

      • "Top to Bottom, Left to Right"

      • "Left to Right, Top to Bottom"

  5. Display the Other tab.

  6. Optionally, change the name of the Data Object.

  7. Click Next >.

  8. Optionally, check Customize the layout of the form?, then click Edit Template to change the default layout of the form.

  9. Click Next >.

  10. Optionally, change the default comment that describes the action.

  11. Click Finish.
