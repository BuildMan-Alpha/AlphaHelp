Setting Object Properties

In this example, we will show you how you can attach scripts to some of the other object events. We will change the appearance of a field label when the corresponding field gets focus. This will highlight the use of the OnArrive and OnDepart events.

To begin the example:

  1. Select the Tables tab on the Control Panel.

  2. Right-click the "Receipts" table and select Open Default Form.

  1. Click the Design button on the toolbar to switch to Design Mode.

Currently all the field labels are bold. We'll start by turning bold off for these labels.

  1. Click on the Customer Id field. Depending on your settings, both the field and its corresponding label may be grouped. In this case both objects will be selected. Since we want to change the property of the Customer Id field's label, click a second time on the Customer Id label. The size handles on the label should be dark, while the size handles on the field should be light (as shown below).

Note : If the field and its label are not grouped, then you can simply click on the Customer Id label.

  1. Right-click Customer Id and select Properties.

  2. Select the Font tab.

  3. Clear Bold.

  4. Click OK.

  1. Right-click on the field label again.

  2. Select Copy appearance from the menu. This copies the appearance properties of the field label to the clipboard.

  3. Right-click the field label again, and select Paste all.

  4. Alpha Anywhere asks whether you want to paste the appearance to all Text objects on the form. Select OK. Your form should now look like this: Picture

  1. Now we will attach a script to the OnArrive event for the Amount field. Select the Amount field. (You may have to click the Amount field twice if it's grouped with the Amount label.)

  1. Right-click and select Events > OnArrive.

The Code Editor appears. Make note of the tab label. "Untitled" is the name of the current form, which is called "Untitled" because we haven't saved it. "Amount" is the name of the current object on the form, and "OnArrive" is the name of the event.

  1. Click Add New Action.

  2. Select the "Object Properties" category, and the "Set Font of Object(s)" Action.

  3. Click OK. The Set Font of Object(s) Genie appears. This screen allows you to set the font properties of one or more objects on a form or browse window. You want to operate on the current form, so leave the selection in the first list box as is.

  4. You need to tell Alpha Anywhere which object on the form we want to manipulate. Click the button for Set ?Font' properties of the following objects text box. A list box of all the objects on the current form appears. Note that the icon to the left of each object indicates the object type.

  1. Select "Amount_lbl". Alpha Anywhere automatically assigned this name to the Amount field label. Your screen should look like this:

  1. Next you need to specify which font properties to set. Click the second button. A dialog box appears displaying which font properties are available for editing.

  1. Select "font.bold", "font.underline", and "font.color". Your screen should now look like this:

For each property you select you have the option of setting the value to:

  • a constant value

  • a value contained in a variable

  • a value that is the result of an expression

  1. You want to explicitly set the properties now, so select the first option, "Specify the value now".

  2. Set the "font.bold" property to "True". Now the field label will be bold when the Amount field gets focus. 

  1. Set "font.underline" to "True".

  1. Set "font.color" to "Light Blue".

  1. Click Next > and Finish to complete the Action. The Code Editor should now look like this:

  1. The asterisk next to the script name indicates that we haven't saved the script. Click the Save button. The asterisk disappears because the script has been saved.

Note : The script is saved in the form, but since the form itself has not yet been saved, if we closed the form without saving it, the script would be lost. This is a very important point to remember.

Now you need to define the OnDepart event for the field. If you did not define an OnDepart event to set the field label back to its original appearance, the field label would change to "blue, underline, bold" when Amount got focus, but would never change back to "black, no underline, normal" when the field lost focus.

To begin defining the OnDepart event, you have two choices. You can either right click on the Amount field again and select Events > OnDepart, or you can click the Select Object? button in the Code Editor. Since you already know how to use the first approach, try the second one here.

  1. Click the Select Object? button.

  2. A list of objects on the form appears. Note that the "Amount_Lbl" object is bold because it's the only object that has events assigned to it.

  3. Select "Amount_Lbl".

  1. Click the Select Event? button. A list of events for the Amount object appears. Note that only the "OnArrive" event is shown in bold because it's the only event that's been assigned a script.

  1. Select the "OnDepart" event. Alpha Anywhere opens a new tab in the Code Editor :

  1. Using the same steps you just followed, select the "Set Font of Object(s)" Action.

  2. Select the "Amount_lbl" object.

  3. Select "font.bold", "font.underline", and "font.color".

  4. Set "False" for "font.bold", "False" for "font.underline", and "Wintext" for "font.color". Your screen should look like this:

  1. Click Next > and Finish to complete the Action.

  2. Click the Save button to save the script.

  3. Click on the form.

  4. Save the form, specifying any name you want.

  5. Click the Form View button to run the form.

  6. Tab into the Amount field, taking note of how the field label changes.

  7. Tab out of the field, taking note of how the field label changes back to its original settings.

While the utility of this script may be limited, the principle is powerful. Action Scripts give you an easy way to manipulate practically any property on a form or browse window, letting you create highly customized user interfaces.
