Table Index Methods

Alpha Five provides the following table methods for manipulating indexes.

Method

Description

<TBL>.INDEX_ADD()

Reuses an existing index by redefining its Order_Expression.

<TBL>.INDEX_CREATE_BEGIN()

Starts the definition of a new index file for a table.

<TBL>.INDEX_CREATE_END()

Creates a new index file for the table.

<TBL>.INDEX_GET()

Returns an object pointer that references an index or query list for the table.

<TBL>.INDEX_NAME_GET()

Returns the tag name of the Nth index tag in the production index for the table.

<TBL>.INDEX_OPEN()

Opens an existing index file, attaching it to an open table.

<TBL>.INDEX_PRIMARY_GET()

Returns an object pointer that references the primary index or query list of the table.

<TBL>.INDEX_PRIMARY_PUT()

Sets the primary index for a table to the specified index tag or query list.

<TBL>.INDEX_RECORD_GET()

Returns an object pointer that references the built-in record number order index tag.

<TBL>.INDEX_TAG_ADD()

Finds an existing index in the production index file for the table.

<TBL>.ORDER()

Orders and filters the records in a table and returns an index pointer.

<TBL>.RELATION_TAG()

Builds an index and returns a handle for the index. It is used in conjunction with the <TBL>.RELATION_ADD() and <TBL>.RELATION_CHANGE() methods.

<TBL>.UPDATE_

PRODUCTION_INDEX()

Updates all of the indexes for a table.
