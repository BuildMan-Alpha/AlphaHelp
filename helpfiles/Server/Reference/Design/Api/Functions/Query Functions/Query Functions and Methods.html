Query Functions and Methods

A query is different from an index. Alpha Anywhere automatically synchronizes an index with table data. A query is static and must be deliberately refreshed to reflect table changes.

Alpha Anywhere provides the following Query methods. Precede each method with the pointer produced by <TBL>.QUERY_CREATE().

Method

 

Description

QUERY.DELETE()

V5

Deletes a saved query.

<QUERY>.DROP()

V5

Deletes the query list file ($$*.MPX) referenced by the object pointer.

<QUERY>.FILENAME_GET()

V5

Returns the full Filename, including drive and path of the query file referenced by the pointer.

<QUERY>.FILTER_GET()

V5

Returns the Filter_Expression for the query list.

<QUERY>.FIND()

V5

Finds a value in the query list and returns a pointer to the Key_Value found.

<QUERY>.FIND_FIRST()

V5

Returns a pointer to the first key value.

<QUERY>.FIND_LAST()

V5

Returns a pointer to the last key value.

<QUERY>.FIND_NEXT()

V5

Returns a pointer to the next key value.

<QUERY>.FIND_PREVIOUS()

V5

Returns a pointer to the previous key value.

<QUERY>.FLAGS_GET()

V5

Returns the index flags (query.options) for the query list.

<QUERY>.ORDER_GET()

V5

Returns the order expression for the query list.

<QUERY>.RECNO_LIST_GET()

V5

Retrieves Count record numbers from the query list.

<QUERY>.RECORDS_GET()

V5

Retrieves the Number_of_Records selected by the query list.

<QUERY>.REFRESH()

V5

Refreshes a query if the table the query is based on has changed.

<QUERY>.TABLE_GET()

V5

Returns a pointer to the table that the query list is based on.

<QUERY>.TYPE_GET()

V5

Returns the type of the index referenced by the object pointer.
