= (Equals Operator)

Syntax

Result as L = Operand1 as A = Operand2 as A

Argument

Description

Operand1

A value or expression of any type.

Operand2

A value or expression of any type.

Description

The Equals operator returns .T. (TRUE) if both Operand1 and Operand2 are of the same type and have the same value. The comparison between character strings is less demanding. The Equals operator removes trailing blanks and ignores case when making a comparison.

Note : You cannot assign the values of a pointer (or dot) variable by using the equals operator. This causes only the addresses of the two pointers to be equal. Use PROPERTY_RECURSE_ASSIGN() to assign the values of one pointer variable to another pointer variable.

Mathematical Assignments

The variable receiving the results of a mathematical calculation is always to the left side of the equals operator.

dim vNum as N

vNum = 5 * 6

? vNum

= 30

vNum = vNum = (5 * 6) / (5 - 6) * -1

? vNum

= 30

Tests of Equality

The following tests of equal were done in the Interactive Window.

? 2 = 3

= .F.

? QTY = 10

= .T. (if QTY equals 10)

? STARTDATE = CTOD("12/18/95")

= .T. ( if STARTDATE contains {12/18/95})

? STARTDATE = DATE()

= .T. ( if the system date equals the date in STARTDATE)

? STATE = "ma"

= .T. ( if STATE contains "MA" )

? LASTNAME = "Ciaccio"

= .F. (if LASTNAME contains "Kalil")

= .T. ( Since the blanks at the end of the strings are automatically trimmed and the matching is not case-sensitive.)
