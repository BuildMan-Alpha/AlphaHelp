Variable Data Types

A variable in the Alpha Anywhere namespace can hold one of the following types of data:

Type Symbol

Type Name

Values

A

Any

The variable can contain any type of data.

B

Blob

Binary data

C

Character

Alpha-numeric characters.

D

Date

A date, where the date is between 00/00/0000 and 12/31/9999. The length is always 8 characters.

F

Function Pointer

Contains a pointer to a function name.

K

Guid

A globally unique identifier

L

Logical

TRUE (.T.) or FALSE (.F.)

N

Numeric

A number of up to 19 digits. There is no concept of a NULL value for a numeric variable or field, which by default has an initial value of 0.

P

Object reference

A reference (pointer) to an object, or a pointer to a "dot" variable.

T

Time

A date/time value that stores the date, hour, minute, seconds and hundredths of a second. The length is always 17 characters.

U

Collection

An ordered list containing any type of data.

Y

Short Time

A time value that stores the hour, minute, seconds and hundredths of a second. The length is always 9 characters.

Z

NULL

NULL value.

Once a variable's data type is determined, it cannot be changed. However, you can use the DELETE command to delete a variable and then use the DIM command to redefine it as a different type.

Note : As a best practice, you should not name variables with the names of data types. For example, do not name a field that contains a date value "Date". However, Xbasic is lenient will not enforce this practice.

Note : There are several table field types, such as exponent numeric ("E"), image reference ("I"), JPEG image ("J"), memo ("M"), OLE ("O"), and rich text memo ("R") that are not data types.
