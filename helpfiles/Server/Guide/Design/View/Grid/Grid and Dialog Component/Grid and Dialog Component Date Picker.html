Date Picker for Grid and UX Components

 

In both the UX Builder and the Grid Builder it is possible to add a Date Picker to your application. The Date Picker includes, among other things, the following features.

  • It can be configured as a date, date/time, or time picker by setting the Date format string.
  • Client-side Event mode allows the first and last date shown in the Date Picker to be dynamically computed using Javascript functions
  • It can be configured to disable certain days of the week. For example, can disable dates on Saturdays and Sundays
To select the new Date Picker in the Grid Builder, make sure that the 'Use Legacy Date Picker property' in the Properties > Miscellaneous section is unchecked. The UX Builder will always use the new Date/Time Picker. To turn a date picker into a date/time picker, simply define a Date Format that includes a time portion.

 


Date/Time Picker properties grid in the UX Builder

 


Additional settings, including localization properties, are found in the Properties > Other  for the UX builder. In the Grid builder these same properties can be found under the Properties page under Miscellaneous.

To create a Date Picker in the UX Builder, click on the Controls button under the design menu. Next go to the Data Controls drop-down under the same menu and select the [Textbox] control. The New Control Dialog will load. Give your control a name and a label, like 'Length of Stay'.  Select the data type for your control, you can choose Character, Date, Time, or Short-time. then click OK to add it to the list of controls.

 

 

 

As noted above, in order to turn the Date Picker into a Date Time Picker it is necessary to change the Date Format setting.

Only one control, way control renders depends on the format string you pass in. If your format string contains a date portion and a time portion you get a date and time picker. If the format string contains only a time portion you will only get a time picker. If the format string contains only a date portion you will end up with only a date picker.

If you go back to UX controls you can override the <Default> setting and set it to what you want in the Date, Date/Time, Time Format Dialog. <DefaultDateFormatOnly>

You can specify low date and high date explicitly, or you can use a javascript function. This can make it possible to prevent users from entering an end date that is before the start date that they entered.

function getLowDate() {

var startDate = {dialog.object}.getValue('START_DATE');

 

if(startDate != '') return startDate;

else return '';

 

}

 

First thing it does is get a variable called startDate and it reads the value that the user typed into the startDate field. And then if the startDate field is not equal to Null than we return the startDate, so now whatever value the user typed into the StartDate field has now been set as the lowdate for the end date field. Otherwise the value is left blank. This makes it easy to dynamically control the start date and end date of your control.

 

Disabled days. If you want to disable, say, Saturdays and Sundays you can enter 0,6 (0 being Sunday and 6 being Saturday) now when you bring up a calander you will see Saturdays and Sundays have been disabled.
