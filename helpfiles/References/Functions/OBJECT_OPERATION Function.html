OBJECT_OPERATION()

Syntax

OBJECT_OPERATION( Operation_Type as C, Operation_Name [, New_Name as C [, Action as C [, Dictionary as C ]]] )

Argument

Description

Operation_Type

The class of operation to act upon. The options are:

  • "append"

  • "bitmap"

  • "browse"

  • "copy"

  • "export"

  • "form"

  • "import"

  • "join"

  • "label"

  • "letter"

  • "mark"

  • "menu"

  • "post"

  • "query"

  • "report"

  • "summarize"

  • "toolbar"

  • "update"

  • "xtab"

  • "script"

  • "function

Operation_Name

The name of the saved operation to process.

New_Name

Optional. Default = "". The name to use when using an Action that requires a second name.

Action

Optional. Default = "delete". The options are:

  • "rename" = renames the object

  • "delete" = delete the object

  • "close" = make a copy of the object in the same dictionary under a new name

  • "move" = copy the object to a different dictionary and delete from current

  • "copyto" = copy the object to a different dictionary

  • "export" = copy the object to a different dictionary - it dictionary does not exists, create it.

Dictionary

Optional. Default = .

Description

The OBJECT_OPERATION() function supports a large number of processes against saved operation. Not all combinations are supported.

Examples

Here is how you would delete an export operation called exportOperationName.

object_operation("export","exportOperationName","","delete")
