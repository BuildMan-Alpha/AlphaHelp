Operation Functions and Methods

Alpha Anywhere provides the following Operation functions and methods.

Method

Description

A5_APPEND_QUICK()

Appends the Transaction table to the Master table.

A5_BATCH_APPEND()

Displays the Batch Append dialog box when a form or browse is open.

A5_BATCH_COPY()

Displays the Batch Copy dialog box when a form or browse is open.

A5_BATCH_DELETE_MARK()

Displays the Delete/Mark/Un-mark dialog box when a form or browse is open.

A5_BATCH_EXPORT()

Displays the Export Records dialog box when a form or browse is open.

A5_BATCH_MOVE()

Displays the Move Records dialog box when a form or browse is open.

A5_BATCH_UPDATE()

Displays the Update Fields dialog box when a form or browse is open.

A5_GET_OPERATIONS()

Returns a list of the saved operations of a specified type.

A5_ADOImportTable()

Imports data from an ADO compatible database or Excel spreadsheet and creates an Alpha Anywhere table.

APPEND.RUN()

Runs a previously defined append operation with the options of running it interactively and returning the Xbasic code that was run.

APPEND.RUN()

Runs a previously defined append operation with the options of running it interactively and returning the Xbasic code that was run.

APPEND.RUN_SILENT()

Runs a previously defined append operation.

COPY.RUN()

Runs a previously defined copy operation with the options of running it interactively and returning the Xbasic code that was run.

COPY.RUN_SILENT()

Runs a previously defined copy operation.

EXPORT.RUN()

Runs a previously defined export operation with the options of running it interactively and returning the Xbasic code that was run.

EXPORT.RUN_SILENT()

Runs a previously defined export operation.

IMPORT.RUN()

Runs a previously defined import operation with the options of running it interactively and returning the Xbasic code that was run.

IMPORT.RUN_SILENT()

Runs a previously defined import operation.

JOIN.RUN()

Runs a previously defined join operation with the options of running it interactively and returning the Xbasic code that was run.

JOIN.RUN_SILENT()

Runs a previously defined join operation.

MARK.RUN()

Runs a previously defined mark operation with the options of running it interactively and returning the Xbasic code that was run.

MARK.RUN_SILENT()

Runs a previously defined mark operation.

NULL_VALUE()

Used in a Global Update Operation, or an Xbasic script to assign a NULL value to a date, numeric, or logical field.

OBJECT_OPERATION()

Performs different types of processes against saved operations.

OPERATION_STARTUP()

Starts the definition of a new Operation by displaying the Create New Operation dialog box.

POST.RUN()

Runs a previously defined post operation with the options of running it interactively and returning the Xbasic code that was run.

POST.RUN_SILENT()

Runs a previously defined post operation.

UI_GET_OPERATION()

Displays a list of operations for a given table/set and operation type.
