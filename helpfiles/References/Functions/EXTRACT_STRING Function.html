EXTRACT_STRING()

Syntax

Sub_String as C = EXTRACT_STRING( Search_in_String as C, Start_Find as C, End_Find as C [, Occurrences as N [, Include_Tags as L [, Case_Sensitive as L [, Regex_Safe as L ]]]] )

Argument

Description

Sub_String

The string extracted from Search_in_String.

Search_in_String

The string to examine for the find text.

Start_Find

The beginning characters of the find text.

End_Find

The ending characters of the find text.

Occurrences

Optional. Default = 1. The nth occurrence to return. The occurrence parameter can be negative, in which case, Alpha Anywhere will return the nth occurrence from the end of the string.

Include_Tags

Optional. Default = .F.

  • .T. = Include the Start_Find and Start_Find characters.

  • .F. = Return only the text between the Start_Find and Start_Find characters.

Case_Sensitive

Optional. Default = .F.

  • .T. = Case sensitive.

  • .F. = Case insensitive.

Regex_Safe

Optional. Default = .F. The "unsafe" characters are ., |, *, ?, +, (, ), {, }, , , ^, $ and \. If your Start_Find and End_Find do not contain any of these characters, you should set Regex_Safe to true to maximize performance.

  • .T. = Improves performance by declaring that the start and end tags are already regular expression safe and EXTRACT_STRING() does not need to escape special characters.

  • .F. = Unsafe characters are replaced. This decreases performance on a large string.

Description

The EXTRACT_STRING() function extracts a sub-string from a string. The sub-string is defined by a starting and ending block of text. The syntax for the function is:

Example

? txt

= this is line one

this is line 2

this is line 3

st = ""

? extract_string(txt, st, en, 1)

= " this is script 1 "

? extract_string(txt, st, en, 3, .t.)

= ""

? extract_string(txt, st, en, -1)

= " this is script 3 "
