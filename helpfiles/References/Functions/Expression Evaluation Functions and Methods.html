Expression Evaluation

Alpha Anywhere provides the following Xbasic expression evaluation functions and method.

Function

Description

A5_EVAL_EXPRESSION()

Evaluates an expression in the context of the current form or browse window.

BETWEEN()

Determines whether the value of Expression is between the values of Lower_Value and Higher_Value.

BETWEEN_CHAR()

Returns a character string that is a quoted filter expression that uses two character values.

BETWEEN_DATE()

Returns a character string that is a quoted filter expression that uses two date values.

BETWEEN_NUM()

Returns a character string that is a quoted filter expression that uses two number values.

BETWEEN_RECORD()

Returns a character string that is a quoted filter expression that uses two record number values.

CASE()

Returns the value ( Value _1 ... Value _16) specified by the first expression ( Expression _ 1 ... Expression _ 16 ) that evaluates to .T. (TRUE).

EVAL()

Evaluates Expression and returns the Result of that expression.

EVAL_KEYLENGTH()

Determines the length of the key specified by Key_Expression.

EVALUATE_TEMPLATE()

Executes the Xbasic code specified by Expression.

EVALUATE_TEMPLATE_

PRIVATE()

Executes the Xbasic code specified by Expression in its own variable name-space.

EVAL_VALID()

Evaluates Expression and returns .T. (TRUE), if Expression is valid (can be interpreted as Xbasic code), or .F. (FALSE) if the Expression is invalid.

IF ... THEN ... ELSE

A conditional control structure, executing the statements for the first logical expression that evaluates to TRUE.

IF()

Evaluates the Logical_Expression. If the expression evaluates to TRUE, the True_Value is returned, otherwise the False_Value is returned.

IIF()

Evaluates the Logical_Expression. If the expression evaluates to TRUE, the True_Value is returned, otherwise the False_Value is returned.

SELECT ... CASE

A conditional control structure comprised of one or more CASE statements. The Logical Expression of each CASE statement is evaluated until a TRUE result is reached. The statements for the TRUE CASE are executed, and program flow continues after the end SELECT.

SELECTLIST()

Returns the value indicated by the Index parameter. The primary use of SELECTLIST()is in Calc Fields. It is easier to use than a group of nested IIF() statements and is simpler to use than a SELECT CASE function.

<SET>.EVAL_VALID()

Evaluates Expression and returns .T. (TRUE), if Expression is valid (can be interpreted as Xbasic code), or .F. (FALSE) if the Expression is invalid.

<TBL>.EVAL()

Evaluates an expression in the context of a table.

<TBL>.EVAL_VALID()

Returns .T. (TRUE) if the expression is valid in the context of the table; otherwise, it returns .F. (FALSE).
