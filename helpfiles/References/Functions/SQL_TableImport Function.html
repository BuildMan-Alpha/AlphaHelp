SQL_TableImport()

Syntax

Status as P = SQL_TableImport( Conn as SQL::Connection, Owner as C, SQL_Table_Name as C, DBF_File_Name as C [, Show_Progress as L [, Size_to_Fit as L [, Allow_Cancel as L [, Add_Table as L [, Status_Format as C ]]]]] )

Argument

Description

Text

A dot variable that contains error information.

Element

 

Description

.HasError

L

Indicates whether an error occurred. The options are:

  • .T.

  • .F.

.ErrorText

C

Contains an error message.

.UserCancelled

L

Specifies whether the user cancelled the operation. The options are:

  • .T.

  • .F.

Conn

A SQL::Connection object that established the connection to the back-end database.

Owner

The owner of the database, if required.

SQL_Table_Name

The name of the back-end database table to copy.

DBF_File_Name

The path and name of the .DBF file to create.

Show_Progress

Optional. Default = .T. . Indicates whether to display a progress indicator on the Status Bar.

Size_to_Fit

Optional. Default = .T. . Indicates whether to size the target table's fields to fit the data.

Allow_Cancel

Optional. Default = .T. . Indicates whether to display a button that allow's the user to cancel the import operation.

Add_Table

Optional. Default = .T. . Indicates whether to create a new table, if it does not already exist.

Status_Format

Optional. Default = "". The format to use to display the Status Bar. This parameter is only relevant if Show_Progress is set to .T. . See STATUSBAR.ABORT_CHECK().

Description

The SQL_TableImport() function imports a table from a back-end SQL database to an Alpha Five .DBF table.

Limitations

Desktop applications only.
