IMPORT_SCRIPTS_UDF()

Syntax

IMPORT_SCRIPTS_UDF( [ Filename as C [, Silent as L [, Automatic_Overwrite as L ]]] )

Argument

Description

Filename

Optional. The name of a text file created with the EXPORT_SCRIPTS_AND_UDF()function. If no filename is specified, then the function prompts for filename.

Silent

Optional. Default = .F.

  • .F. = Alpha Five displays a dialog box showing the scripts and UDFs in the text file that you are importing. You can select which scripts and UDFs to import.

  • .T. = This dialog is suppressed, and Alpha Five imports all scripts and UDFs.

Automatic_Overwrite

Optional. Default = .F.

  • .F. = Alpha Five does prompts before overwriting any script or UDF with the same name.

  • .T. = Alpha Five does not prompt before overwriting any script or UDF with the same name.

Description

The IMPORT_SCRIPTS_UDF() function imports scripts and user defined functions from a text file.

Note : The IMPORT_SCRIPTS_UDF() function is called when you use the Import command on the right click menu from the Code tab on the Control Panel. It allows you to import scripts and UDFs into a database that were previously exported using the Export command (also on the right click menu from the Code tab on the Control Panel ). Using this function, it is possible to write code that allows you to update your customer's applications by sending them enhanced scripts and UDFs.

Example

export_scripts_udfs("c:\myfunctions.txt")

import_scripts_udfs("c:\myfunctions.txt")
