COUNT_TEXTBLOCKS()

Syntax

Count as N = COUNT_TEXTBLOCKS( Search_in_Text as C, Start_Find_Text as C, End_Find_Text as C [, Case_Sensitive as L [, Regex_Safe as L ]] )

Argument

Description

Count

The number of instances of the find text in Search_in_Text.

Search_in_Text

The string to examine for the find text.

Start_Find_Text

The beginning characters of the variable length find text.

End_Find_Text

The ending characters of the variable length find text.

Case_Sensitive

Optional. Default = .F.

  • .T. = Case sensitive

  • .F. = Case insensitive

Regex_Safe

Optional. Default = .F. The "unsafe" characters are ., |, *, ?, +, (, ), {, }, , , ^, $ and \. If your Start_Find_Text and End_Find_Text do not contain any of these characters, you should set Regex_Safe to true to maximize performance.

  • .T. = Improves performance by declaring that the start and end tags are already regular expression safe and COUNT_TEXTBLOCKS() does not need to escape special characters.

  • .F. = Unsafe characters are replaced. This decreases performance on a large string.

Description

The COUNT_TEXTBLOCKS() function returns the number of instances of the variable length Find_Text found in Search_in_Text.

Example

txt = <<%txt%

this is line one

this is line 2

this is line 3

%txt%

st = ""

? count_textblocks(txt, st, en, .t.)

= 0.000000

? count_textblocks(txt, st, en, .f.)

= 3.000000
