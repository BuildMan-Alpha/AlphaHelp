FTP_PUT_FILE()

Syntax

Result as C = FTP_PUT_FILE( Address as C, User as C, Password as C, Source_File as C, Destination_File as C [, Modal as L [, Title as C ]] )

Argument

Description

Result

When Modal is .F., the Result value will be "OK", "ERROR" or "Cancel", depending on the result of the file transfer.

Address

The FTP address of the transmission.

User

The user's identity.

Password

The user's password.

Source_File

The file to send.

Destination_File

The file to save remotely.

Modal

Optional. Default = .T.

  • .T. = The displayed dialog is modeless.

  • .F. = The dialog is modal, and the script pauses till the FTP transfer is complete.

Title 

Optional. Default = "FTP File Transfer". The title of the dialog box.

Description

The FTP_PUT_FILE() function sends a file in the background over a FTP connection.

Note : The FTP protocol uses 2 ports for uploading and downloading files, port 21 and some random high-numbered port.

When publishing via FTP, Alpha Five first opens a connection to port 21 on the remote server. This port is used for exchange of control information and for negotiating the second port to use. The second port is chosen by Windows and will be some value above 1024. This is the port used for the actual transfer of files. This is all standard FTP behavior.

Alpha Five has no control over the second port used. Norton Antivirus apparently looks for activity on certain ports and flags any traffic on those ports as activity by a trojan. Phineas Phucker uses port 2801. Striker uses port 2565. WinCrash uses ports 2583, 3024, 4092 and 5742. These are all perfectly acceptable for use by an FTP upload.

Supported By

Alpha Five Version 6 and Above

Example

ftp_put_file("www.alphasoftware.com", "alpha5", "password", "c:\learningcenter.asp","/technical_documentation/learningcenter.asp")
