XBASIC_ASYNCHRONOUS()

Syntax

XBASIC_ASYNCHRONOUS()

Description

The XBASIC_ASYNCHRONOUS() function causes the script to wait until after other Alpha Five processes have been completed, not including screen refreshes.

Note : XBASIC_WAIT_FOR_IDLE() waits for the main loop to idle. The main loop is the entire Alpha Five application. For example, if there are any messages that need to be processed (such as a screen repaint), the Xbasic script will wait until those messages have been processed. So for example, this will allow a screen repaint, a focus change, initializing of menus and toolbars on a  form open, etc. to happen before your Xbasic script continues. <br> <br> UI_YIELD(), which was added in Version 7, is similar to XBASIC_WAIT_FOR_IDLE() , but just allows pending screen repaints to happen. It does not allow other messages (such as form activate, deactivate, change of focus etc) in the message queue to happen.

In the context of Version 7 XBASIC_ASYNCHRONOUS()actually does not do anything. In Version 6 XBASIC_ASYNCHRONOUS()forced a thread that was running in foreground to run in the background. But in Version 7 there is no foreground or background thread; there is just a single thread. The function remains to prevent Xbasic scripts from failing.

In the context of Version 7 XBASIC_SYNCHRONOUS()does nothing. In Version 6 it forced a script to run in the main thread.

Limitations

Desktop applications only.

Example

' start process

xbasic_asynchronous()

' start another process
