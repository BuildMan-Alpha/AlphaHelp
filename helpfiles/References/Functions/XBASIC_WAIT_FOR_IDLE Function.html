XBASIC_WAIT_FOR_IDLE()

Syntax

XBASIC_WAIT_FOR_IDLE( [ Wait_Time as N ] )

Argument

Description

Wait_Time

The number of seconds to wait.

Description

The XBASIC_WAIT_FOR_IDLE() function causes the script to wait for other Alpha Five processes and screen refreshes to complete before proceeding.

The function forces each action to complete before the next action starts. If code is written that requires one action to complete before the next begins, then the XBASIC_WAIT_FOR_IDLE() would be appropriate. An example is a succession of operations where the result of one operation, such as an append, must be completed before an update operation.

Note : XBASIC_WAIT_FOR_IDLE() waits for the main loop to idle. The main loop is the entire Alpha Five application. For example, if there are any messages that need to be processed (such as a screen repaint), the Xbasic script will wait until those messages have been processed. So for example, this will allow a screen repaint, a focus change, initializing of menus and toolbars on a  form open, etc. to happen before your Xbasic script continues. UI_YIELD(), which was added in Version 7 , is similar to XBASIC_WAIT_FOR_IDLE() , but just allows pending screen repaints to happen. It does not allow other messages (such as form activate, deactivate, change of focus etc) in the message queue to happen.

In the context of Version 7 XBASIC_ASYNCHRONOUS()actually does not do anything. In Version 6 XBASIC_ASYNCHRONOUS()forced a thread that was running in foreground to run in the background. But in Version 7 there is no foreground or background thread; there is just a single thread. The function remains to prevent Xbasic scripts from failing.

In the context of Version 7 XBASIC_SYNCHRONOUS()does nothing. In Version 6 it forced a script to run in the main thread.

Limitations

Desktop applications only.

Example

' start process

xbasic_wait_for_idle()

' start another process
