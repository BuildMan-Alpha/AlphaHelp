Syntax

Argument

Description

Expression

Any valid expression that includes a field or a combination of fields from the current table or set that specifies what you want to count. You can also specify a constant value, such as 0 or 1.

GRP � >Group

Optional. When to use the function (instead of on every record).

The Group and Sub_Group are optional parameters that define the range of records to summarize, and may or may not be required, depending upon the context of the summary calculation. Possible values for GRP � >Group and GRP � >Sub � Group include:

  • GRAND (all records)

  • The name of the current table.

  • The name of a table in the current set.

  • The name of a grouping level that is defined in a report layout.

  • DETAIL (over the records in the detail section)

GRP � >Sub � Group

Optional. When to reset the value produced by the function to 0 (instead at the end of all records).

Description

COUNT() returns the number of records in a group of records. COUNT() is equally valid in a calculated field on a form. COUNT() can be used in reports, and on forms to summarize data across multiple records.

For example, to count the line-items in an invoice set, the GRP � >Group parameter might be GRP � >INVOICE and the GRP � >Sub_Group parameter GRP � >INV_ITEM.  This means the summary operation includes only those child records in INV_ITEM related to the current parent record in the INVOICE table.  To summarize all the records in a table or set, use the name GRAND as the Group name. The Group GRP � >GRAND and Sub_Group GRP � >INV_ITEM produce a summary result based on all invoices.

Limitations

This function is a report writer function, not intended for table level field rules or other expressions. While the function may perform in some areas outside of the report writer, its use there is not supported.
