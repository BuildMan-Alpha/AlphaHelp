Define an ADO Table or View Query

This procedure is part of the Grid Builder procedure for building a component.

Establishing the Connection

  1. Select Grid > Query (ADO).

  2. In the Connection type list select "ADO Connection String".

Using a Named Connection String

A connection string specifies which database you are connecting to and how you will login to it. If you do not have any existing connection strings, continue with Specifying a Connection String.

  1. Select "Used Named Connection String" from the list to the right of the Connection type list.

  2. Select the connection string from the Connection Name list.

  3. Click Connect. If the Grid Builder was able to open the database, additional fields will appear on the dialog box.

Specifying a Connection String

If you need to create a connection string:

  1. Select "Specify Connection String" from the list to the right of the Connection type list.

  2. Optionally, click to enter the connection string. Click OK when you are finished.

  3. Optionally, click Build to display the Data Link Properties dialog box (this is a Windows operating system dialog).

  4. Consult your database provider or database administrator for the required settings.

  5. Click OK to continue.

  6. Click Connect. If the Grid Builder was able to open the database, additional fields will appear on the dialog box.

  7. Select how to retrieve data. The options are:

    • Table or View

    • SQL Select Statement

Retrieving Your Data From a Table or View

If you selected "Table or View" in step 7:

  1. Click to select a table or view from the Table/View list.

Note : A table alias can make your grid portable, by separating it from direct reference to a table name. as long as a table has the same structure and the same alias, it can use a grid originally built for another table.

  1. Optionally, enter an alias for the table into the Table/View alias field. If you do not enter an alias, it will default to the name of the table.

  2. Optionally, specify a filter expression in the Filter field that selects records from the table. Click to display the Specify Filter dialog. Refer to Specifying a Filter Expression.

  3. Optionally, specify a ordering expression in the Order by field that sequences selected records. Click to display the Specify Order dialog. Refer to Specifying an Order Expression.

  4. Optionally, check Read only to prevent user updates. If you are in the process of creating a grid, continue with Selecting the Fields to Display.

  5. Click Select Primary Key(s) to display the Specify Primary Keys dialog. Select the field(s) that uniquely identify each record and click OK.

Retrieving Your Data with a SQL Statement

If you selected to retrieve your data with a "SQL Select Statement":

  1. Select SQL Select Statement.

  2. Click Edit SQL to display the SQL Builder to define the table(s) and fields that you want to work with.

  3. Continue with SQL Builder.
