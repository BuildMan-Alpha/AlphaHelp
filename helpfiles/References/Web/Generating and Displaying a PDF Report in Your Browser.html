Generating and Displaying a PDF Report in Your Browser

If you look at the source of the PrintInvoice.A5W page, included in the Web Applications Demo, you will see the following HTML and Xbasic code.

<html>

<head>

<meta name="generator" content="Alpha Five HTML Editor">

<title></title>

</head>

<body>

<p>

<%A5

dim filter as C

dim order as C

dim filename as C

IF eval_valid("invoiceNumber") = .f. Then

    goto skipreport

end if

Filter = "invoice_number = " + quote(invoiceNumber)

Order = ""

filename = Session.Session_Folder + chr(92) + "tempreport.pdf"

filename = report.saveas("Invoice@[PathAlias.ADB_Path]\invoice.set", "pdf", filter, order, filename, .f.)

if file.exists(filename)

    Response.Redirect(Session.Session_Url + "tempreport.pdf?" + time("hms3"))

end if

end

skipreport:

?"Error: You must call page with a query string that specifies the InvoiceNumber.<br>" + crlf()+\

"For example: " + Request.Script_Name + "?invoiceNumber=000001<br>"

%>

</p>

</body>

</html>

Of interest are the following lines of Xbasic code.

filename = report.saveas("Invoice@[PathAlias.ADB_Path]\invoice.set", "pdf", filter, order, filename, .f.)

if file.exists(filename)

    Response.Redirect(Session.Session_Url + "tempreport.pdf?" + time("hms3"))

The first line uses the REPORT.SAVEAS() method to print the Invoice report of the Invoice set. Note several things:

  • When writing a normal script for an Alpha Anywhere application, the report would be identified as  some_path\Invoice@invoice.set. In this case, we have added [PathAlias.ADB_Path]. This is a placeholder that Alpha Anywhere will fill with the actual value of some_path when the page is published.

  • The REPORT.SAVEAS() method is producing a PDF version of the report, saving it to the specified filename.

Forcing the Browser to Load the New PDF Instead of Using a Cached Copy

The second line of Xbasic code tests whether the specified file was written to disk. If it was written:

  • The code uses RESPONSE.REDIRECT()to instruct the browser to open the report file.

  • The URL of the report is a combination of Session.Session_Url (the URL prefix used by a client browser to retrieve a temporary file), tempreport.pdf (the specified filename), ? (which allows the following value), and time("hms3") (which produces a nearly random value).

The extra time string makes the page different. It is a trick to force the browser to load the file, rather than showing a file it might have in its cache. You will notice that when the Web Component Builder does a live preview it also does this. Otherwise the user might have to click the Refresh button in the browser after the browser is opened before the correct page is shown.

The page is not limited to opening a file under the webroot. In this case however, the PDF file is a temporary file. When the Application Server creates a new session, it defines a temporary folder that it will delete when the session goes away. By putting temporary files in this folder, you can let the Application Server take care of any cleanup automatically.

Displaying the PDF Report in a Different Window

By default, the URL that calls PrintInvoice.A5W displays the PDF report in the same window. To cause the PDF report to display in a different window, the link should be modified as follows.

<a href="PrintInvoice.A5W" target="_blank">Print this</a>

Then, the following statement would display the PDF report in the new window.

Response.Redirect(Session.Session_Url + "tempreport.pdf?" + time("hms3"))
