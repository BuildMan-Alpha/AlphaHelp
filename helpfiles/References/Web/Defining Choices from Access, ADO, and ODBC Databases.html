Defining Choices from Access, ADO, and ODBC Databases

Dialog components may have controls that present the user with a list of choices. When you wish to dynamically populate a choice list from an Access or ADO/ODBC compatible database, you need to use the SQL SELECT dialog box.

When first opened the SQL SELECT dialog box displays the SQL Genie tab. You have the choice of using this tab or the SQL SELECT tab to define the SQL SELECT statement that retrieves the choices that will populate the control.

Using the SQL Genie

  1. Select a table or view from the Table list box.

  2. Select a field from the Field to display list box.

  3. If you want to store a different value than you display:

    1. Check Store value other than the display value.

    2. Select a field from the Field to store list box.

  4. If you would like to display only a subset of the available records:

    1. Check Filter data.

    2. Select a field from the Define Filter Field list box.

    3. Select an operator from the Operator list box.

    4. Choose whether the field will be compared to a "Variable" or "Constant".

    5. If the Value field appears, enter the value of the constant.

    6. If the Variable Name field appears, select a variable from the list box.

  5. If you have defined a filter expression, continue on the Parameter Mapping tab.

  6. Click OK to finish or Cancel to discard your inputs.

Using the SQL SELECT Tab

  1. Enter a SQL SELECT statement that retrieves the choices that will populate the control.

  2. If the control's stored value is the same as its displayed value, then the SQL SELECT statement should return a single field. For example: SELECT CustomerName FROM Customers ORDER BY CustomerName.

  3. If the control's stored value is different from its displayed value, then the SQL SELECT statement should return a two fields. The first field is the display value and the second field is the stored value. For example: SELECT CustomerName, CustomerID FROM Customers ORDER BY CustomerName.

  4. If the SQL SELECT statement includes a WHERE clause, you can use a ? as a placeholder in the statement to reference variables (or constant values). SELECT CustomerName, CustomerID FROM Customers WHERE City = ? ORDER BY CustomerName. You can then specify how variables map to each ? in the WHERE clause.

  5. If you list displays duplicate values, use the DISTINCT keyword in your query. For example: SELECT DISTINCT city FROM Customers ORDER BY City.

  6. Always include an ORDER BY clause that orders the query on the first field (i.e. the displayed value).

  7. You may click the SQL Builder button to use the <span class=Screen>SQL Builder</span> utility. Note : This utility will overwrite the contents of the SQL SELECT tab.

  8. If you have defined a filter expression, continue on the Parameter Mapping tab.

  9. Click OK to save your inputs or Cancel to quit.

Parameter Mapping

If you have defined a filter on the SQL Genie or the SQL SELECT tabs, the SQL expression will contain a "?" that refers to a variable or constant. The Parameter Mapping tab will appear.

  1. Select each parameter that appears in the Specify variables/values list box.

  2. Optionally, change the mapping selection in the Parameter maps to list box.

  3. Optionally, change the name of the variable in the Variable name list box.

  4. Optionally, change the value in the Constant value field.

  5. Select the data type of the constant or variable in the Parameter type list box.
