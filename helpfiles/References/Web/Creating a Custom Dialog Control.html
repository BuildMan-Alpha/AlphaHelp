Creating a Custom Dialog Control

A custom control allows you to fully define the content of a textbox, label, or other type of dialog control. The content is produced by a function that typically processes a combination of Xbasic and HTML code. Other than the requirement that the function return valid HTML, there are no other important constraints upon what the grid cell can display.

To create a custom control:

  1. Display the Form Controls page of the Dialog Builder.

  2. Select the control that you want to modify in the Selected Controls list.

  3. Change its Control Settings > Control Type to "Custom".

  4. Click the or button in the Custom Control Properties > Definition field to display the Define Xbasic for Custom Control dialog. This dialog allows you to define the code that produces the return value of a function. The function is named controlname_render(), where controlname is the name of your control.

The function will return a value that is composed of quoted HTML and unquoted Xbasic.

Variables

You may dimension local variables of any type within the fieldname_render() function. However, if you want to create variables that have scope beyond the current grid cell, you must define it as an element of either the:

  • args pointer (valid for the dialog page)

  • session pointer (valid for the session)

Field Names

At any time you can insert the name of any of the controls on your dialog into the code by clicking Insert > Data > FieldName.

System Variable

The Define Xbasic for Custom Control dialog provides the following system variable that you can use in your calculations.

Name

Type

Description

IsPostBack

L

.T. if the dialog has been submitted back to itself one or more times, and .F. the first time the dialog is run.

To insert this variable into your code, click Insert > Args > IsPostBack.

A Custom Control Example

A custom control where Xbasic dynamically sets font and background properties, depending on whether the quantity value is above or below the average.

In the following example, the field name is Phone. as a result, the function is named Phone_Render().

function Phone_render as c (data as p, args as p, session as p)

dim temp as C

on error goto Phone_xbasicError

select

    case ("617" $ Data.Phone)

        temp = <<%html%

%html%

    case ("781" $ Data.Phone)

        temp = <<%html%

%html%

    case else

        temp = <<%html%

%html%

end select

temp = temp + <<%html%

size="20" maxlength="20"class="SquareHipInput" type="TEXT" name="PHONE" value="%html%

phone_render = temp + Data.Phone + <<%html%

" >%html%

end

Phone_xbasicError:

Phone_render = "Error in custom control xbasic: " + error_text_get()

end function

In this example, the select ... case clauses have similar content. Note how the value of the control is refreshed with a reference to the phone control, using the prefix " data. ".

Although not a required technique, embedding the HTML inside the <<%html% ... %html string simplifies the coding by removing the requirement to escape special characters.
