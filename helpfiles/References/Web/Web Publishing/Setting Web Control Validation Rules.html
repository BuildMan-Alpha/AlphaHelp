Setting Web Control Validation Rules

Field validation is defined in a login control's User ID Options > User ID Validation Rules and Password Options > Password Validation Rules properties.

Note : Some validation rules are limited when defining User ID and Password validations.

After submission of the page data, if any validation errors are found, Alpha Five does not commit any of the edits. Instead, it returns a page describing all of the errors.

All of the page data has to be valid before any data is committed. For example: a grid has 10 rows and a user makes edits in rows 1, 2 and 3, but validation errors are only found in row 1. Alpha Five does not commit ANY of the edits (even though, in theory, it could have committed the edits in row 2 and 3 since there were no validation errors found in those rows).

Note : If you have a Grid that shows both existing data and new rows, the new rows are not validated unless there is at least one value in the new row that has been changed from its initial value. Here is why this is done: Suppose you have set an "Is Required" rule for a field in the Grid. Assume you have a Grid that shows both existing data and 3 new rows. When the user makes an edit to a field in an existing row, if Alpha Five validated the new rows (which don't contain any edits), a validation error would occur because the three new rows do not have values in the field that is required. There is one exception to this rule. If the Grid is set to "Add records only", and the number of new record rows is set to 1, then the new record row is always validated when it is submitted, regardless of whether the user has edited any fields in the row.The Field Validation dialog box allow you to control the data that a user enters into an editable field on a dialog.

After defining the field validation rules that you want, enter test values in the Value to test field and click Test. The Results field will indicate whether the test allowed or disallowed the value.

Saving and Loading Validation Rules

Alpha Five version 7 allows you to save the validation rules of a grid component as a named file, and to load validation rules from one of these files.

To save validation rules:

  1. With the Field Validation dialog open, click Save/Load Rule Sets.

  2. Enter a name for the validation rules and click Save.

To load validation rules:

  1. With the Field Validation dialog open, click Save/Load Rule Sets.

  2. Select a set of validation rules and click Load.

Require Value

The most commonly used validation rule is "Require Value". If you set the control to not require a value, this control can test user input with a ((|#Custom_expression|custom expression)) or accept a NULL value.

Range Checking

The most commonly used ((|#Custom_expression|custom expression)) is a range check in the format.

currentValue >= low_value .AND. currentValue <= high_value

Remember, that character strings (even if composed of number characters) are evaluated on alphabetic order.

Validating the Uniqueness of 2 or More Fields

The following script is placed in the CanInsertRecords event script of the grid. It assures that the combination of firstname and lastname fields is unique.

dim tbl as P

dim indx as P

 

query.filter = "firstname = " + quote(dataSubmitted.firstname) + ".and. lastname = " + quote(dataSubmitted.lastname)

tbl = table.open("PathAlias.ADB_Path\customer")

indx = tbl.query_create()

if (indx.records_get()> 0) then

    Result.Cancel = .t.

    Result.ErrorHTML = "Duplicate record"

end if

tbl.close()

Validation Expression Dialog Box

  1. Enter an expression that evaluates to a true (logical) value. Use the variable CurrentValue to represent the value that the user has entered. Remember that CurrentValue is a character variable, even though it may be representing a date or numeric value.

  2. Optionally, enter a value that should trigger either a true (.T.) or false (.F.) evaluation into the Test value field and click Test.

  3. Click OK to continue or Cancel to discard your inputs.

Regular Expressions Dialog Box

  1. Optionally, select an existing regular expression from the list and click OK. (The list that you see is loaded from a file named a5_regex.txt and located in the Alpha Five program folder.)

  2. Optionally, create a new regular expression.

    1. Select "".

    2. Enter the name of the expression in the Name field.

    3. Enter the regular expression in the Regular Expression field.

    4. Click OK to save your input.
