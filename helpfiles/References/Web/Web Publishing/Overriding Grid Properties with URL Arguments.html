Overriding Grid Properties with URL Arguments

When you place a grid component on an .A5W page, you have always been able to override any of the settings in the grid component using Xbasic code. Now you can also override settings of the grid component from the URL that displays the page containing the grid component. For example, say that you have a page called page1.a5w that contains a grid component called myGrid. The following URL will display the page:

http://localhost/page1.A5W

Now, let's say that when you display the page you want to override the CSS style that was specified when the grid was designed, and instead, use the "Jazz" style. The following URL will accomplish this:

http://localhost/page1.A5W?myGrid_osc_style_name=jazz

The general syntax for overriding a property from the URL is:

< ComponentAlias > _ OS< PropertyType > _ < PropertyName > = < NewValue >

Argument

Description

The alias of the component. This is the alias that was assigned to the component when it was placed on the .A5W page. This is not necessarily the same as the component name.

A single character designating the data type of the property you want to override.

  • "c" = character

  • "n" = numeric

< PropertyName >

The name of the property that you want to override.

The new value of the property.

You can override as many of the Grid properties as you want. Just include a name/value pair in the URL for each property that you want to override.

Note : All variables created as part of a URL are of type character.

Example

To change the style, and the number of rows:

http://localhost/page1.A5W?myGrid_osc_style_name=jazz&myGrid_osn_rows=5

Supported By

Alpha Five Version 7 and Above

Limitations

Web publishing applications only.
