Creating Navigation System Components

Overview

You can use Navigation System toolbar components to add a menu to an A5W page. Menus can have all entries on a single level, or can have child entries, which in turn can have child entries. Menus can have either a horizontal or vertical layout. The display of a list of child entries can be caused by either clicking on or by moving the mouse over the parent entry. Here are examples of a simple, two level menu using two different cascading style sheets.

The process of creating and using navigation components can be broken down into these steps.

  1. Create your pages.

  2. Define the way that you want to navigate between these pages.

  3. Create a navigation component that reflects this definition.

  4. Add a two cell table with zero width lines to each page (an invisible table).

  5. Move the page content that will be below or to the right of the navigation component into the bottom or right-most cell of the table.

  6. Add an A5W_INCLUDE() statement to load the navigation component into the top or left-most cell of the table.

Defining Toolbar Properties

To create a navigation component.

  1. Start the Web Projects Control Panel.

  2. Click New on the toolbar.

  3. Select Web Component and click Next >.

  4. Select Navigation System and click OK.

  5. Select the Navigation Type. The options are:

    • "Horizontal Toolbar"

    • "Vertical Toolbar"

  1. Select a cascading style sheet from the Style list.

Note : Navigation System Components can only use the CSS style sheets delivered with Alpha Five.

  1. Select Navigation System in the menu.

  2. Optionally, change the Root Settings > Menu value from "menu" to a more meaningful name.

  3. Optionally, change the Root Settings > Display Method value. This is the same value set in step 1.

  4. Determine how you want the menu selection to operate in the Root Settings > Menu Event field. There are two options.

    • "onclick" - which loads the new page when the user clicks the menu selection

    • "onmouseover" - which loads the new page when the user moves the mouse over the menu selection

  5. Optionally, change the default width of the toolbar in the Root Settings > Width field. Text that exceeds this length will wrap to multiple lines. If the width value is empty, menu entries will be immediately adjacent to each other. If the width is 0, menu entries will wrap to the length of the longest word.

  6. If you are creating a Vertical Toolbar, set the Root Settings > Inner Width field. This value must be equal to or smaller than the Root Settings > Width field.

  7. Optionally, change the default width of the toolbar in the Root Settings > Height field. If the height value is empty, the text of the menu entries will define the height of the toolbar.

  8. Optionally, select a cascading style sheet from the Root Settings > Style field. This is the same value set in step 2.

Adding Toolbar Entries

  1. Click Add Child to add the first menu entry.

  2. In the Node Settings > Type field select the content of the menu entry. The options are:

    • "Link" - a connection to another page

    • "HTML" - freeform HTML text

    • "Separator" - only for vertical toolbars

  3. If the choice in step 2 was "Link":

    1. Optionally, change the access level in the Node Settings > Security field. Users with security access levels greater than or equal to this value will be able to see the menu entry. Users with security access levels lower than this value will not be able to see the menu entry.

      The security access level is compared to a protected session variable named session.protectedSecurityLevel. Your application code should set this variable to reflect the access rights of the current user.

    2. Optionally, if the entry has child entries, enter one or more rules in the Node Settings > Button highlight rule field. These rules cause the menu entry to highlight when specified pages are displayed. Refer to Creating Button Highlight Rules.

    3. Optionally, enter the text to display in the Node Settings > Text field.

    4. Optionally, click in the Node Settings > Icon field to select an image to display from the <span class=Screen>Select Image</span> dialog box. If you want to add an expression to the end of the URL, see Overriding the Component's Style Sheet.

    5. If Click in the Node Settings > url field to select the page to show when the user clicks the menu entry.

    6. If a menu entry haschild menu entries below it, the Node Settings > Fallback URL field will appear instead. This optional field specifies the page to show if no child entries are selected.

    7. Optionally, change the Node Settings > Target field to define the window to use for the page that you open. Leave the entry blank to use the current window.

    8. Optionally, change the Node Settings > Width field to override the default width setting.

    9. Optionally, change the Node Settings > Target field to specify the name of the window which will display the page specified in the Node Settings > url or Node Settings > Fallback URL fields.

      • "" = open the page in the current window.

      • "_blank" = open the page in a new window

  4. If the choice in step 2 was "HTML":

    1. Enter any HTML code that you wish to display in the toolbar entry.

    2. Click in the Node Settings > HTML field to display the HTML Editor. Input and edit the HTML text that you wish to display, then click Save.

    3. Optionally, set a width value in the Node Settings > Container Width field. This value overrides the value set in the Root Settings > Width field.

  5. Repeat steps 1 through 4 to add additional menu elements.

Creating Page Highlight Rules

When a menu entry has child entries, you may specify button highlight rules. A button highlight rule causes the parent menu entry to be highlighted when a specified page is opened. This feature only works when you use the technique described in Placing Navigation System Components.

The page name may be specified exactly or you may use wildcard characters to specify a range of similarly named pages that are located under the same menu entry.

To use button highlight rules:

  1. Select a parent menu entry that has child entries.

  2. Click in the Node Settings > Button highlight rules field to display the Button Highlight Rules dialog. The Project Pages list is for reference only. It lists the pages in your project.

  3. To add a page name or pattern click Add to display the Add Pattern dialog.

  1. Enter a full page name or a partial page name, using "*" as a wildcard. The "*" character matches multiple characters.

  2. Click Test Pattern to see which pages will cause the parent menu entry to highlight. In the example above, opening three different pages (which are under the same parent menu entry) will cause the parent menu entry to be highlighted.

  3. Click OK to define the button highlight rule.

Setting Access Levels

Each child menu has a Node Settings > Security value. Each user with an access (security) setting greater than or equal to this value will be able to see the menu entry. Each user with a access setting less than this value will not be able to see the menu entry.

This is the same property used to control access to fields in a grid (see also Select the appropriate display format ) and grid update features (see Permissions ).

Note : A user with a security level of 0 has unrestricted access. Any user without an assigned security level has a security level of 0.

Using Navigation Components

There are two ways to use navigation components.

  • You can place the component directly on the page (described below in Placing Navigation System Components ).

  • Use an A5W_INCLUDE() statement to load the navigation component into another page as it is displayed (described below in Loading Navigation System Components ). This is a more flexible approach that allows you to add navigation to finished pages without substantial editing. This approach is more flexible and easier to maintain.

Placing Navigation System Components

  1. Save the page.

  2. Click Insert Component > Select, pick a component, and click OK. This will display the Insert Component dialog box.

  3. Optionally, select a menu entry from the Selected Button list. This will be the selected menu entry when the navigation component loads.

  4. Optionally, clear the Container check box. When cleared, the builder will place only the Xbasic code required to place the navigation component. This option allows you to place the navigation component at any location on a finished page.

  5. If you leave the Container check box checked, you will see that the component is occupying one of the cells of a table with only two cells.

  1. Drag all the other content of the page (components, text, images, etc.) into the empty cell. In the case of a horizontal navigation component, the page content will be in the bottom cell. In the case of a vertical navigation component, the page content will be in the right cell.
