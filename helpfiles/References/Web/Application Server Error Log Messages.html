Application Server Error Log Messages

Message

Description

"Connection reset by peer"

Means that when the server tried to read the request from the client, the client was no longer connected. This is typically caused when the user clicks their Stop button in their browser, but can also be caused by general Internet connection issues. This error condition itself is not abnormal. However it could be indicative of the server running slowly and users giving up waiting for pages to load.

"Read timed out"

Indicates that the server was trying to read a request from a client and that client failed to send a complete HTTP request within the allotted timeout period. This happens in one of 2 ways - the client does not send the full headers of the request within 90 seconds or the client does not send the full request, including any body, within the 5 minutes following the receipt of the headers. This can be caused by a very slow Internet connection or general Internet congestion, but they almost always are a sign of a denial of service (DoS) attack being run against your server.

"Connection timed out"

Means that a read timed out. A connection was established with a client, but that client did not send at least a single byte within 10 seconds. This can be caused by a very slow Internet connection or general Internet congestion, but they almost always are a sign of a denial of service (DoS) attack being run against your server.
