A5W Page HTML FORM Syntax

Syntax

<FORM NAME=" Variable " METHOD=" Method " ACTION=" Directive "> <INPUT_1><INPUT_2>...<INPUT_N> </FORM>

Argument

Type

Description

Variable

C

The name of the variable to pass to the page named by the ACTION Directive.

Directive

C

Defines where to send the data collected by the form. Values can be:

  • a URL

  • " <%A5 ? Request.script_name %> " = sends form input back to the HTML page.

Method

C

Defines how to send the form's data to the server.

  • "Get" = You may save (bookmark) and refresh URLs formatted through a HTTP GET command. Some devices may restrict the total length of a URL to 128 characters, which may trim the arguments appended by a GET command.

  • "Post" = Used for forms that submit a large amount of data. There is no length restriction with POST commands. You cannot bookmark the arguments sent by a HTTP POST command.

<INPUT_1>

<INPUT_2>...

<INPUT_N>

C

One or more <INPUT>, <SELECT>...</SELECT>, or <TEXTAREA>...</TEXTAREA> statements.

Description

A form is basically a collection of <INPUT>, <SELECT>, or <TEXTAREA> controls with NAME and VALUE attributes. These controls accept data from the user. When you click the SUBMIT button, the browser collects the stored values, associates them with their names, and (usually) sends the resulting data to the server. The METHOD attribute defines how to transmit the data to the server. The ACTION attribute specifies how the data is processed at the server.

Refer to an HTML reference for a complete description of the <FORM> tag.

Example

<FORM ACTION="<%A5 ? Request.script_name %>">

<INPUT TYPE="SUBMIT" NAME="a_dlg_button" VALUE="First">

<INPUT TYPE="SUBMIT" NAME="a_dlg_button" VALUE="Previous">

<INPUT TYPE="SUBMIT" NAME="a_dlg_button" VALUE="Next">

<INPUT TYPE="SUBMIT" NAME="a_dlg_button" VALUE="Last">

<INPUT TYPE="HIDDEN" NAME="this_recno" VALUE="<%A5 ? tbl.recno()%>">

</FORM>
