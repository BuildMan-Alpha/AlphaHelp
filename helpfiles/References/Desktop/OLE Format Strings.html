OLE Format Strings

Functions which accept format strings, process those strings on a character by character basis. as each character is processed, it is replaced with the value indicated in the tables below. If a character is not recognized it is passed to the output string unchanged.

For example:

If the format is "N" + chr(9) + "|" + "H", the output string will contain the name followed by a tab and then a vertical bar and finally the name of the help file.

There are three groups of formatting character sets based on the type of function being called.

Groups of Format Character Sets

Supported Formats

Class: Formatting of classes, enumerated types and interfaces

D - Documented Name

H - Help File Name

I - Documentation String\n" \

K - Type Kind

N - Name

L - Type Library File

U - Globally Unique Identifier

X - Help File Context Id

Constant: Formatting on constant values

C - Type Name

D - Documented Name

I - Integer Value

H - Help File Name

K - Invocation Type Kind

L - Type Library File Name

N - Name

V - Documentation String

X - Help File Context Id

Method: Formatting of methods and properties

A - Class Name

D - Class Documented Name

H - Help File Name

I - Documentation String

K - Method Invocation kind

L - Type Library Name

M - Prototype string without return type

N - Method Name

P - Prototype string with return type

R - Return type

U - Globally Unique Identifier of the Class

V - Type Kind of the Class

X - Help File Context ID
