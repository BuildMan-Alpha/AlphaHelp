Logical AND

When a logical AND (.AND.) is placed between two logical expressions, the result is TRUE only if the logical expressions on both sides of the .AND. are TRUE.

When a logical OR (.OR.) is placed between two logical expressions, the result is TRUE if one of the expressions on either side of the .OR. is TRUE.

For example, if Sales contains 2000 or State is "MA,":

If Sales contains 2000 and State is "CA," or if State contains "MA," regardless of the value in Sales:

(SALES > 1000 .AND. STATE = "CA") .OR. STATE = "MA"  ->  TRUE

Logical NOT

Unlike .AND. and .OR., the logical NOT operator (.NOT.) does not go between two logical expressions. Rather, it reverses the logical state of a single expression.

For example, if Sales is 900 and State is "NY":

.NOT. (3 > 2)  ->  FALSE

.NOT. (3 > 4)  ->  TRUE

.NOT. (SALES > 800)  ->  FALSE

.NOT. (SALES > 1000 .AND. STATE = "CA") .OR. STATE = "MA"  ->  TRUE
