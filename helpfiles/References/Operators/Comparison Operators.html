Comparison Operators

Comparison operators compare two expressions which must be of the same type (either character, numeric, or date) and will yield a logical result (i.e., TRUE or FALSE). Any expression involving a comparison operator is called a logical expression.

Comparison operators are typically used in creating search criteria, filters, or are used with the IF and CASE functions. Alpha Five supports the following comparison operators:

Exactly Matching

The result is TRUE if the expressions on both sides of the operator are equal. The Exactly Matching operator does not automatically remove trailing blanks. The two strings must match exactly, character for character, by case and by length (including blank spaces).

Note : When comparing a character field to a string, remember that the length of the field is pre-defined. If the field is assigned fewer characters than its defined length, the remainder of the field is padded with spaces.

"Foley" == "Foley    " -> .F.

Since the number of trailing blanks are different

"Foley" == "foley" -> .F.

Since the case is different

Not Exactly Matching

The result is TRUE if the expressions on both sides of the operator are not equal. The Not Exactly Matching operator does not automatically remove trailing blanks.

Since the case of the characters does not match

Since the blanks at the end of the strings are not trimmed

!$

Substring Exclusion

This operator determines whether the character expression on the left of the operator is not contained anywhere within the character expression on the right. Operations involving the !$ operator return a logical result (i.e., TRUE or FALSE). The string comparison is not case-sensitive.

"Computer" !$ COMPANY -> .F.

"computer" !$ COMPANY -> .F.

"computer" !$ LOWER(COMPANY) -> .F.

If COMPANY contains "The Computer Store"

$

Substring Inclusion

This operator determines whether the character expression on the left of the operator is contained anywhere within the character expression on the right. Operations involving the $ operator return a logical result (i.e., TRUE or FALSE). The string comparison is not case-sensitive.

"Computer" $ COMPANY -> .T.

"computer" $ COMPANY -> .T.

"computer" $ LOWER(COMPANY) -> .T.

If COMPANY contains "The Computer Store"

All comparison operators, except for substring inclusion, can evaluate numeric, date, or character values. For date values, earlier dates have lower values. For character values, the character's corresponding ASCII value is used in the comparison.
