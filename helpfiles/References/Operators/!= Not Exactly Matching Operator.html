!= (Not Exactly Matching Operator)

Syntax

Result as L = Operand1 as A != Operand2 as A

Argument

Description

Operand1

A value or expression of any type.

Operand2

A value or expression of any type.

Description

The Not Exactly Matching operator returns .T. (TRUE) if both Operand1 and Operand2 are of different types or have different values. If two character strings have different characters, different case, or a different number of trailing blanks, then Not Exactly Matching operator will return .T. (TRUE).

The opposite of "!=" is "==" ( Exactly Matching Operator ).

Note : When comparing a character field to a string, remember that the length of the field is pre-defined. If the field is assigned fewer characters than its defined length, the remainder of the field is padded with spaces.

Examples

"Foley" != "Foley    " -> .T. since the number of trailing blanks are different

"Foley" != "foley" -> .T. since the case is different
