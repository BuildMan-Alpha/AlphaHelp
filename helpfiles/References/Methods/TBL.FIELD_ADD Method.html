Description

The .FIELD_ADD() method is used between the TABLE.CREATE_BEGIN() and TABLE.CREATE_END() methods to define the fields that form a table structure. The first field of the table is always defined with TABLE.CREATE_BEGIN(), subsequent fields are defined with .FIELD_ADD().

A field definition consists of at least two parameters, the Field_Name and Field_Type, and can contain up two optional parameters, Width and Decimal Width. Field_Type is a single-character code representing one of the field types. The chosen field type determines how the optional parameters are used. If the field type is Character, an additional Width parameter is supplied, and if the field type is Numeric, the additional Width and optional Decimal_Width parameters are specified. Date, Memo, Rich text memo, Bitmap, OLE, and Logical fields are always defined with two parameters because the field widths for these field types are set automatically.

Supported By

Alpha Five Version 5 and Above

Example

Create a table.

table.create_begin("firstname","c",30)

table.field_add("lastname","c",30)

table.field_add("price","n",5,2)

table.field_add("quantity","n",2)

table.field_add("birthday","d")

'The create_end method creates the table and assign an object pointer to the table

tbl = table.create_end("sample.dbf")
