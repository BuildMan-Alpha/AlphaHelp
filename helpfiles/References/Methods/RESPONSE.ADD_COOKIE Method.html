RESPONSE.ADD_COOKIE()

Obsolete
This method is obsolete and has been removed from the software as of Version 11. See Replacing Response.Add_Cookie in V11.


Syntax

RESPONSE.ADD_COOKIE( Name as C, Value as C, Lifetime as N )

Argument

Description

Name

The name of the cookie.

Value

The information that the cookie contains.

Lifetime

The length of time to save the cookie in seconds. Setting the lifetime to 0 creates what is known as a session cookie. Session cookies expire as soon as the user closes the web browser and they are not stored on the user's hard drive as other cookies are.

Description

The RESPONSE.ADD_COOKIE() method creates a cookie. The server for a page will automatically send all unexpired cookies to the client's browser. Each cookie that is available to the page will create a variable named Name containing Value.

Note : Response variables are local to the page (script) and must be passed to any functions defined on the page.

Example

This example creates a cookie named "Application" which holds the value "Alpha Five" and will stay on the client for 2592000 seconds (6 months).

response.add_cookie("Application", "Alpha Five", 2592000)

This example creates a session cookie named "username" which holds the value "jdoe" and will be removed as soon as the user closes the browser.

response.add_cookie("username","jdoe",0)

This script tests to see if the cookie "myname" exists. If the cookie exists, the script displays the cookie's value. If the cookie does not exist, the script creates it.

<html>

<head>

<meta name="generator" content="Alpha Five HTML Editor">

<title></title>

</head>

<%a5 if eval_valid("request.cookies.myname") then

    ? "<P>The cookie myname is "+request.cookies.myname+"</P>"

else

    response.add_cookie("myname","greg",0)

end if

? "<form action=\""+request.script_name+"\" method=\"post\">"

%>

<P>The first time through, this will set the cookie "myname" to "greg"</P>

<input type="submit" value="Press to submit"> </form>

<body>

</body>

</html>
