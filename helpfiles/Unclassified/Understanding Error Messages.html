Understanding Error Messages

When you have an Xbasic programming problem on an A5W web page, you will see an error like the following.

500 Internal Server Error

Script Error

Error:Script: /UserTable.a5w line:22

x_UserTable = a5w_run_Component(tmpl_UserTable)

Property not found

tmpl.request.variables.v.r2 subelement not found.

The following table describes the various parts of the error message.

Message

Description

500 Internal Server Error

This is the HTTP protocol's specification for a script error. It is included so that the Application Server can adhere to the HTTP/1.1 specification.

Script Error

Xbasic is saying that the error classification was a script error. This is a bit redundant given the preceding line in the message, but it is included so that you do not need to be familiar with the HTTP protocol in order to determine what type of error occurred.

Error:Script: /UserTable.a5w line:22

The error was on line 22 of /UserTable.a5w.

x_UserTable = a5w_run_Component(tmpl_UserTable)

Line 22 of /UserTable.a5w contains the Xbasic code shown above.

Property not found

The general error was that a property being referred to could not be found.

tmpl.request.variables.v.r2 subelement not found

Specifically, tmpl.request.variables.v.r2 is the property (sub-element) that could not be found.

In plain English, the code in a5w_run_Component() refers to tmpl.request.variables.v.r2, but tmpl.request.variables.v.r2 does not exist.

Limitations

Web publishing applications only.
