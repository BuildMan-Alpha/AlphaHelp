The SQL syntax that the Alpha Five Enterprise Edition and Application Server supports is first constrained by the SQL syntax supported by the target database. The Application Server has additional requirements.

The following is a simplified SQL grammar syntax as supported by the Alpha Five Portable SQL parser. Support for these constructs depends on the target database and its implementation of SQL.

The grammar that follows shows the components of the grammar in the general format:

Argument

Syntax

Argument

Alternative | Alternative [ Optional_Value ]

Notes on Syntax

  1. Words beginning with lower case are rules.

  2. Words in all capital letters are literals or key words and appear as typed. The actual select statement is case insensitive.

  3. The vertical bar ( | ) indicates that any of the alternatives are valid. Commas are part of the input stream.

  4. Items in square brackets [] are optional.

  5. With some exceptions, parentheses shown are a required part of the input stream.

  6. The FROM clause of a SELECT statement contains both TableName and AliasName, as in the following statement:

SELECT Customers.CompanyName, Customers.ContactName, Customers.ContactTitle FROM Customers Cust

  1. The Application Server does not support the "SELECT *" syntax to select all fields from a table. You must reference the table name, as in "SELECT TableName.*".

Delete Syntax

The Portable SQL DELETE statement deletes one or more records from the specified table.

Note : The Application Server does not support DELETE statements.

Object to Delete

Syntax

Record(s)

DELETE FROM Table_Name WHERE Column_Name = Value_Expression

Update Syntax

The Portable SQL UPDATE statement modifies one or more records in the specified table.

Note : The Application Server does not support UPDATE statements.

Object to Update

Syntax

Record(s)

UPDATE Table_Name SET Column_Name = Value_Expression_1 WHERE Column_Name = Value_Expression_2

Insert Syntax

The Portable SQL INSERT statement adds a record to the specified table.

Note : The Application Server does not support INSERT statements.

Object to Insert

Syntax

Record

INSERT INTO Table_Name ( Column_Name_1, Column_Name_2, ... , Column_Name_N ) VALUES ( Value_Expression_1, Value_Expression_2, ... Value_Expression_N )
