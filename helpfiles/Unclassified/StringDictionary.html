StringDictionary

Syntax

Description

Xbasic supports the Collection data type. Collections are fine for relatively small sets of values. However, the StringDictionary object offers similar features, but is much faster than collections when used for very large lists. The StringDictionary object only supports character data.

Note : Although collections are slower, they are more flexible, because they can store any data type.

Methods

The StringDictionary object supports these methods:

Method

Type

Description

<StringDictionary>.CLEAR()

V

Removes all keys from the string dictionary.

<StringDictionary>.EXISTS()

L

Determines if an item exists in the dictionary.

<StringDictionary>.GET()

C

Gets a value from a dictionary.

<StringDictionary>.INITIALIZE()

V

Add keys to a dictionary.

<StringDictionary>.REMOVE()

V

Removes a key from a dictionary.

<StringDictionary>.SET()

V

Sets a key value in a dictionary.

Example

dim sd as stringdictionary

 

sd.Set("One","1")
sd.Set("Two","2")
? sd.Get("One")
= "1"
? sd.Get("Two")
= "2"
