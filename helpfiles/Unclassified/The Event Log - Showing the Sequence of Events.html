The Event Log - Showing the Sequence of Events

It is easy to look at each object and field rule and see which events can be triggered. However, knowing the order in which these events are triggered when used in combination is not as straightforward.

Alpha Five provides a tool, called the Event Log, for exploring events and the order in which they occur. When the log is on, a message appears in the Trace window describing each field rule event or object event as it occurs.

To Turn On/Off the Event Log :

  • To turn on the Event Log, open the Trace window from the View menu and check the box.

  • To turn off the Event Log, clear the box in the Trace window.

  • To see the order in which events are triggered, turn on the Event Log. Then open a form or browse window and navigate, enter, edit, and save records. You will see each event listed in the Trace window.

Note that the Trace window allows you to distinguish between field rule events and object events.

Field rule events are listed in the Trace window as follows:

Event: Field(table alias->field name) "event name"

'For example

Event: Field(customer->lastname) "CanEditField"

Object events are listed in the Trace window as follows:

Event: Event: object address "event name"

'For example

Event: customer:last_name "OnArrive"
