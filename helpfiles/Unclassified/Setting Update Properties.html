Setting Update Properties

This topic continues to describe the process of creating a grid component.

The Grid Update Settings page only appears if the grid is editable (updateable).

Permissions

  1. Select Grid > Update Settings. The resulting dialog boxes control the editing properties of the component.

  2. If you want to prevent the user from deleting records, uncheck Permissions > Allow delete.

  3. If you want to prevent the user from editing records, uncheck Permissions > Allow update.

  4. If you want to prevent the user from inserting records, uncheck Permissions > Allow insert.

  5. Optionally, change the number of empty new record rows that will appear at the bottom of the grid from 3 in the Permissions > Number of new record rows field.

  6. If you want to allow the user to delete records, optionally change the Permissions > Delete security level. Users with a delete security level greater than or equal to this number will be allowed to delete records. If the user is logged on with insufficient Security Levels for delete, then the Delete check box in the grid and the Delete button in the Detail View will not be displayed.

  7. If you want to allow the user to update records, optionally change the Permissions > Update security level. Users with a update security level greater than or equal to this number will be allowed to edit records. If the user is logged on with insufficient Security Levels for update, delete and insert, then the 'Submit' button on the Grid will not be displayed.

  8. If you want to allow the user to insert records, optionally change the Permissions > Insert security level. Users with a insert security level greater than or equal to this number will be allowed to insert records. If the user is logged on with an insufficient Security Level then the 'Add' button is in the Detail View will not be displayed.

Validation

  1. Optionally, uncheck Validation > Only validate edited fields (V7). Unchecking this field will speed the process of your page.

  2. Optionally, change the way errors will be reported in the Validation > Template for reporting errors field. The following placeholders are available.

    • {Field} - the name of the field that had the error

    • {Value} - the value that was placed into the field

    • {Error} - the error message returned by the Application Server.

  1. Optionally, change the instruction message that appears in the Validation > Instruction message field when an error occurs.

  2. Optionally, change the error page row heading in the Validation > Error page 'Row' heading field.

  3. Optionally, change the error page error heading in the Validation > Error page 'Error' heading field.

Customization

  1. Optionally, change the submit button label in the Customization > Submit button label field.

  2. Optionally, change the delete check box label in the Customization > Delete check box label field.

  3. Optionally, select the Customization > Has 'Cancel' button check box. This selection allows you to set the next property.

  4. Optionally, change the cancel button label in the Customization > 'Cancel' button label field.

  5. Optionally, select the Customization > Confirm before submit check box. This selection displays a confirmation dialog box and allows you to set the next property.

  6. Optionally, change the message that appears in a confirmation dialog box in the Customization > Confirm message field.

  7. Optionally, click in the Customization > Target page field to display the Specify Target Page dialog. Make a selection and click OK. This page appears if the update is successful. The options are:

  8. Specify a full URL

  9. Specify to return to this page

  10. Select an .A5W or .HTM page from your project

Conflict Checking

  1. Optionally, select "True" (clear the check box) in the Conflict Checking > Check for write conflicts list. When selected, before the record changes are applied the Application Server makes sure that no other user has saved changes to this record.

  2. If you are checking for write conflicts, specify the type of checking to perform in the Conflict Checking > Write conflict check type field.
