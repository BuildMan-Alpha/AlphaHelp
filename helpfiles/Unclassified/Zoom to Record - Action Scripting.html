Zoom to Record - Action Scripting

Overview

The Zoom to Record action opens the form view of the current record in an embedded browse. This action is useful for quickly seeing more details on a record.

When initiated, the Zoom to Record action opens a Form displaying the record that has focus in the embedded browse. In order for the action to work, a record in the embedded browse must have focus.

Procedure

  1. Select whether to specify the target form in advance or prompt the user at run-time. The options are:

    • "Prompt user for name of target Form at run-time"

    • "Specify target Form now"

  2. If you selected "Specify target Form now", select the form in the Form Name field.

  3. Optionally, click Filter forms to limit the Form Name list to those belonging to a single table or set.

  4. De-select the If target Form is already open, switch to it? check box if you always want to open a new instance of the target form.

  5. If you de-selected the If target Form check box, optionally check the Open target Form as Dialog check box. The dialog box will be modal, meaning that the user must close it before moving on to any other task.

  6. Click Next >.

  7. Optionally, change the pointer name in the Pointer field.

  8. Optionally, change the scope of the pointer variable. Scope controls what other scripts can use this pointer.

    • "Session" means that all the other scripts on the window you opened will have access to the variable.

    • "Global" will allow any script running, even from another form, to access this pointer.

  9. Click Next >.

  10. Optionally modify the comment that describes the action.

  11. Click Finish.
