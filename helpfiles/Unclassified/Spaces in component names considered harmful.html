Spaces in component names considered harmful

If you create a component in Alpha5 V10 that has a space or spaces in its name, the IDE will change the spaces to underscores when saving the component. This is because Web pages sometimes have a problem loading a component that has a space in its name.

Unfortunately, the IDE has no control over changes you make in Windows Explorer or in a command console. If you add a space to a component name, the application server may have a problem loading and displaying the component for users.

In addition, be careful if you rename components; you also need to change the component name in any page or component that uses the renamed component.
