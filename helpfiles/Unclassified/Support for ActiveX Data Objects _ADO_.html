Support for ActiveX Data Objects (ADO)

Alpha Five provides ADO (ActiveX Data Objects) support that enable interactive connections with popular back-end SQL database.

ADO is a widely used Microsoft technology allowing access to relational databases (such as  SQL Server, Oracle, MySQL, Access .mdb files) and to other non-relational data sources (such as  Excel and text files).

Alpha Five has extensive support for ADO in the following areas: Xbasic scripts and Xdialog forms can display and update data from any ADO data source. You can also build web applications using the Application Server that display and update data from an ADO data source.

The following Xbasic code snippet shows how you could populate an array called lastname with information from a table:

dim name[1] as C

dim conn as P

dim rs as P

dim sql as C

dim connection_string as C

conn = ole.create("adodb.connection")

rs = ole.create("adodb.recordset")

connection_string = "Provider=SQLOLEDB.1;Integrated Security=SSPI;Persist Security Info=False;Initial Catalog=customers;Data Source=SQLSERVER"

sql = "select lastname from customer where state = 'ma' "

conn.ConnectionString = connection_string

conn.Open()

rs.Open(sql,conn)

while .not. rs.eof()

    name[] = rs.fields.item("Lastname") 'add name to end of array

    rs.moveNext()

end while

conn.close()

delete rs

delete conn
